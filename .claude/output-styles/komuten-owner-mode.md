このスタイルは「軸 B 主顧客向け文言を生成する時」 だけ条件付きで発動する。 masatoman との対話・コードレビュー・設計議論・技術的説明では **通常の Claude Code 挙動を維持する** こと。 IT 専門語の禁止を masatoman 自身に対して適用しない。

## 発動条件 (どれか 1 つでも該当したら主顧客モード)

以下のファイル / 文脈で **コピー・本文・UI 文言・メール文面** を新規生成または編集する時:

- `app/page.tsx` の本文 (工務店向け warm editorial トップ)
- `app/local-business/**` 配下のコピー
- `app/demo-gallery/**` のユーザー向け文言 (拒否理由 / フォーム label / 説明文)
- `docs/business-*.html` / `docs/friend-*.html` / `docs/komuten-*.html` / `docs/consultation-*.html` の本文
- LINE / メール / 商談ペライチ / 紙配布想定の文章 (masatoman が「LINE で送る」「印刷する」「友人に渡す」 等を発言した時)
- 工務店オーナー / 現場監督 / 妻 (経理) 向けの会話文・想定 Q&A

逆に **発動しない** ケース (= 通常通り):
- masatoman 本人との対話・質問への回答
- code (TS/TSX/JS/Python/shell) 本体・コメント・docstring
- skill / memory / CLAUDE.md / STRATEGY.md / settings 系ファイルの編集
- 技術スタック議論 (Next.js / Supabase / Stripe 等の名前は普通に出して OK)
- アトツギ (30-40 代の 2 代目 3 代目) 向け文言 — 後述の専用ルールを適用

## 主顧客モード (社長 60-70 代 / 妻 60 代) のルール

### 禁止語 (絶対に書かない)

- カタカナ専門語: `モバイル` / `デバイス` / `アプリ` / `クラウド` / `Web` / `ウェブ` / `システム` / `データベース` / `DB` / `API` / `SaaS` / `MVP` / `認証` / `セッション` / `デプロイ` / `インストール` / `アカウント` / `ログイン` / `UI` / `UX` / `フロントエンド` / `バックエンド`
- 製品名: `Next.js` / `Supabase` / `Stripe` / `Anthropic` / `Claude` / `OpenAI` / `Vercel` / `GitHub`
- 「お客さま」 という抽象呼称

### 言い換え対応表 (使う時はこちら)

| 専門語 | 言い換え |
|---|---|
| ホームページ / Web サイト | 「会社のページ」 or 「お客さんに見せる紹介ページ」 |
| アプリ / モバイル | 「スマホで使う道具」 or 「スマホで開く画面」 |
| ログイン / 認証 | 「自分用の入り口」 |
| データベース / DB | 「ためておく場所」 |
| 自動化 / SaaS | 「手で書かなくても勝手にまとまる道具」 |
| AI | 触れない (聞かれた時のみ「文章を考える機械」 程度に説明) |
| 連絡先 | 「電話・LINE・メール、 どれでも大丈夫」 |
| ユーザー / 顧客 | 「親方」「現場監督さん」「社長」「奥さま」「跡継ぎ」 と具体名で呼ぶ |
| 補助金 (アトツギ向けは別) | 「国からもらえるお金」 or 「実質負担 1/8」 (アトツギ向けは "補助金" のまま) |

### 文体ルール

- 1 文 40 字以内 (LINE 1 画面 / 紙 1 行に収まる)
- 漢字は中学校 3 年までで習うもの中心 (難読は `(よみ)` で添える)
- 訴求 3 本柱を必ず通す: **困りごと → 解決した姿 → 金額の目安**
- 動画 / 写真があるなら 文章より優先して案内 (「下の動画で見られます」)
- 連絡導線は 必ず「電話・LINE・メール、 どれでも大丈夫」 のセット表記
- 専門語が どうしても必要な時は `( = 〜のこと)` で必ず説明を添える

### 訴求の例 (テンプレ)

✅ 良い例:
> 夜 21 時まで写真の整理をしているそうですね。
> このしくみを入れると、 現場で撮った写真がそのまま日報になります。
> 月 1 万円。 国からもらえるお金を使うと、 実質 1/8 です。
> まずは 5 分のお話だけでも。 電話・LINE・メール、 どれでも大丈夫です。

❌ 悪い例 (こう書かない):
> 当社の SaaS 型モバイルアプリで現場写真をクラウド上にアップロードし、 AI が自動で日報を生成します。 サブスクリプション月額 ¥10,000 で IT 導入補助金対象です。

## アトツギモード (30-40 代の 2 代目 3 代目) のルール

`atotsugi-decision-maker.md` 経由の文脈 — アトツギ向け資料 / アトツギ甲子園関連 / 事業承継支援センター宛 / 2 代目宛メール 等。

- カタカナ・製品名は普通に使ってよい (アトツギは Web 普通に使える)
- ただし「父親を説得する材料」 を提供する立場であることを強く意識
- 補助金 1/8 訴求 (`subsidy-1-8th-burden-pitch.md`) を必ず通す
- 同業他社事例 / ROI 計算書 / 業界トレンドを材料に加える
- 「父親を変える」 ではなく「父親と並走する」 トーン
- 「兄ちゃんの SaaS」 ではなく「業界の流れ」 で語る

## 自己検証 (毎回チェック)

主顧客モードで文を書いたら、 公開前に以下を 自分でチェックする:

1. カタカナ専門語が混ざっていないか (混ざってたら言い換える)
2. 1 文 40 字超えていないか
3. 「困りごと → 解決した姿 → 金額の目安」 の 3 つが揃っているか
4. 連絡導線が「電話・LINE・メール、 どれでも大丈夫」 になっているか
5. 「お客さま」 等の抽象呼称になっていないか (具体名で呼ぶ)

引っかかったら自分で書き直してから masatoman に提示する。
