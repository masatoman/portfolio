# タスク完了時のチェックリスト

## コード品質

### 1. TypeScript型チェック
```bash
pnpm tsc --noEmit
```
- [ ] 型エラーがないことを確認
- [ ] `any` 型の使用を最小限に抑える
- [ ] 適切な型定義がされている

### 2. リンティング
```bash
pnpm lint
```
- [ ] ESLintエラーがないことを確認
- [ ] 警告も可能な限り解決

### 3. コードの可読性
- [ ] 変数名・関数名が意味を持っている
- [ ] 複雑なロジックにコメントがある
- [ ] DRY原則（Don't Repeat Yourself）に従っている
- [ ] KISS原則（Keep It Simple, Stupid）に従っている

## 機能テスト

### 4. 手動テスト
```bash
pnpm dev
```
- [ ] 新機能が期待通りに動作する
- [ ] 既存機能が壊れていない（リグレッションテスト）
- [ ] エラーハンドリングが適切
- [ ] ローディング状態が適切に表示される

### 5. レスポンシブ対応
- [ ] デスクトップ（1920px以上）で表示確認
- [ ] タブレット（768px〜1919px）で表示確認
- [ ] モバイル（320px〜767px）で表示確認
- [ ] Chrome DevToolsのレスポンシブモードで確認

### 6. ブラウザ互換性
- [ ] Chrome（最新版）で動作確認
- [ ] Safari（最新版）で動作確認
- [ ] Firefox（最新版）で動作確認
- [ ] Edge（最新版）で動作確認

## パフォーマンス

### 7. ビルド確認
```bash
pnpm build
```
- [ ] ビルドエラーがない
- [ ] ビルド警告を確認・対応
- [ ] バンドルサイズが適切

### 8. 画像最適化
- [ ] `next/image` コンポーネントを使用
- [ ] 適切なサイズの画像を使用
- [ ] WebP/AVIF形式の使用を検討
- [ ] alt属性が適切に設定されている

### 9. コード分割
- [ ] 不要なコンポーネントの動的インポートを検討
- [ ] 大きなライブラリの遅延読み込みを検討

## アクセシビリティ

### 10. 基本的なアクセシビリティ
- [ ] キーボードナビゲーションが機能する
- [ ] フォーカス状態が視覚的に分かる
- [ ] セマンティックHTMLを使用
- [ ] ARIA属性が適切（Radix UIで自動対応）

## セキュリティ

### 11. セキュリティチェック
- [ ] ユーザー入力のバリデーション
- [ ] XSS対策（Reactの自動エスケープ）
- [ ] 環境変数の適切な管理
- [ ] 機密情報がクライアントサイドに露出していない

## 文書化

### 12. ドキュメント更新
- [ ] 新機能の説明を追加（必要に応じて）
- [ ] READMEの更新（必要に応じて）
- [ ] コメントの追加（複雑なロジック）
- [ ] TODOコメントの確認と対応

## Git操作

### 13. コミット前の確認
```bash
git status
git diff
```
- [ ] 意図した変更のみがステージングされている
- [ ] デバッグ用のconsole.logが削除されている
- [ ] 不要なコメントアウトされたコードが削除されている
- [ ] 環境変数やAPIキーが含まれていない

### 14. コミットメッセージ
```bash
git commit -m "適切なコミットメッセージ"
```
- [ ] 変更内容が明確
- [ ] 変更理由が分かる（必要に応じて）
- [ ] 日本語または英語で統一

## デプロイ前

### 15. 最終確認
```bash
pnpm build
pnpm start
```
- [ ] 本番ビルドが正常に動作
- [ ] 環境変数が適切に設定されている
- [ ] 外部サービス（Resendなど）が正常に動作
- [ ] エラーログがない

### 16. パフォーマンス確認
- [ ] Lighthouseスコアを確認
- [ ] Core Web Vitalsが目標値内
  - LCP < 2.5秒
  - FID < 100ms
  - CLS < 0.1

## YOLOモード（自動テスト実行）

### 17. 自動実行（推奨）
タスク完了後、以下を自動的に実行：
```bash
# TypeScript型チェック
pnpm tsc --noEmit

# リンティング
pnpm lint

# ビルド確認
pnpm build
```

これらが成功すれば、基本的な品質は保証されます。

### 18. タスク完了後の振り返り
- [ ] 目標は達成できたか？
- [ ] より良い実装方法はなかったか？
- [ ] 学んだことは何か？

## 注意事項

### 優先順位
1. **必須**: TypeScript型チェック、リンティング、ビルド確認
2. **推奨**: 手動テスト、レスポンシブ対応、アクセシビリティ
3. **任意**: パフォーマンス最適化、ドキュメント更新

### YOLOモードでの実行
自動テストを優先的に実行し、手動確認は必要最小限に抑えることで、開発速度を維持しながら品質を保証します。
