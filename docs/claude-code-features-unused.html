1Output Styles最有力推し

/config → Output style で切替。 CLAUDE.md / skill とは別レイヤーで system prompt 自体を書き換える 機能。

公式組み込み 3 種

名前挙動
Proactive確認を待たず即実行 (auto mode 相当、 permission prompt は残る)
Explanatoryコード実装中に「Insights」 解説を挟む
LearningTODO(human) マーカーで一部実装を masatoman に任せる学習モード

masatoman さん向け活用案

  • 「demo-gallery 投票分析モード」 custom output-style — 票数より拒否理由分布を主軸に解釈するペルソナを default 化
  • 「工務店オーナー向け説明モード」 — IT 用語禁止 + LINE で送れる文に変換
  • .claude/output-styles/komuten-mode.md で project 単位に固定可能
skill との違い: skill は呼び出し型、 output-style は 毎ターン自動適用。 主顧客モードを永続化したい時に強い。 
---
name: Komuten Owner Mode
description: IT 用語禁止 + LINE 配布想定の平易日本語に変換
keep-coding-instructions: true
---

回答は工務店オーナー (40-60 代、 LINE はできるが Web 入力は苦手) に送る前提で書く。
- カタカナ専門語禁止 (「モバイル」「アプリ」 等)
- 1 文 40 字以内
- 操作手順は番号付き、 1 ステップ 1 動詞
- 専門語が必要な時は ()内に「= 〜のこと」 を必ず添える

2Hooks の多彩なイベント

masatoman さんは RTK (token 削減 hook) を使っているが、 hook イベントは 20+ 種類 ある。 軸 B プロジェクトで効くもの:

Hook活用案
UserPromptSubmitプロンプト送信前に「これは軸 B / 軸 A どっち?」 を STRATEGY.md と照合して警告
PostToolUse (Edit/Write)demo-gallery / 主要 LP 編集後に Prettier + Playwright スモークを自動実行
UserPromptExpansion/x-post-rules や /masatoman-article-rules 起動時に追加 context を強制注入
SubagentStopExplore/Plan agent 終了時に dev_journal に作業記録を自動 append
CwdChangedportfolio ↔ 個人開発マネタイズプラットフォーム の往復で env を切り替え
Stopturn 終了時に session-recap 候補が溜まったか自動判定
PreCompact圧縮直前に重要な決定を memory に逃がす保険
特に UserPromptSubmit hook で軸 B/A 自動判定 は STRATEGY.md の主軸切り替えと相性が良い。

3Web 版 Claude Code + iOS app

terminal CLI 中心だと推測されるが、 web (claude.ai/code) / iOS は こういう時に使える:

  • 外出先で 商談 / ヒアリング メモを起こす → portfolio repo に反映 (5/22 北原氏面談、 友人ヒアリング当日)
  • 長時間タスクを kick off して放置 → 戻ったら結果を terminal に claude --teleport で引き取り
  • 出先で「issue-finder の結果見たい」 が手元の Mac なしで完結
  • iOS app は Slack 連携 で @Claude メンション → PR が返ってくる workflow も組める

4Remote Control / Dispatch / Channels

機能挙動
Remote Control自宅 Mac で動かしてる session を スマホから操作。 外出中に「あの skill だけ叩いて」 が可能
Dispatchスマホから 1 行投げる → Desktop app が session を自動生成して処理
ChannelsTelegram / Discord / iMessage / 自前 webhook から session に event push

軸 B の友人ヒアリング当日、 LINE 経由で「この回答どう解釈する?」 を即投げる経路として Channels (LINE は無いが iMessage 経由) が刺さる可能性。

5Worktree 隔離で並列実験

isolation: "worktree" を agent に渡すと、 一時 git worktree で作業して 終わったら自動クリーンアップ。

  • demo-gallery の 拒否理由 UI を A/B 並行検証 したい時 (1 つは本番ブランチ、 1 つは worktree)
  • issue-finder の profile 拡張 (FP / micro-corp / it-subsidy) を 互いに干渉せず 4 並列実行
何も commit しなければ worktree は自動消滅するので 思いつき実験コストが激安。

6Routines (cloud) vs /loop vs Desktop scheduled tasks

masatoman さんは /schedule と /loop を skill で持っているが、 3 つの違いを使い分けると効く:

種類動く場所用途
Routines Anthropic クラウド Mac 落ちても動く / GitHub event / API trigger 可 (sns-automation の本番化候補)
Desktop scheduled tasks masatoman 自身の Mac ローカル file 直接触りたい時 (issue-finder の supabase 直接書き込み等)
/loop 現セッション内 一時的なポーリング (商談ペライチ生成中の進捗監視等)

Routines は sns-automation の API 課金回避ルートに化ける可能性 (Claude Code MAX 内蔵 web_search を cloud で動かせる)。

7/fast モード

/fast で Opus のまま出力速度を上げる (model を downgrade しない)。 ペライチ HTML 生成や 大型 review で体感差が大きい。 masatoman さんは Opus 4.7 (1M context) なので即使える。

8fewer-permission-prompts skill available

system prompt 内に出ているが、 まだ実行してない可能性が高い。

  • 過去 transcript を scan して 安全な read-only コマンドを .claude/settings.json に自動 allowlist 追加
  • bq / gh api / supabase 系の毎回の permission prompt が消える
  • RTK と併用可

軸 B フェーズで Supabase MCP / Notion MCP を頻繁に叩くので 効果が大きい。

9simplify skill available

変更コードを review して 再利用可能性 / 品質 / 効率 の観点で問題があれば修正まで自動でやる。

  • demo-gallery の vote-form.tsx 等 直近大量編集箇所に走らせると 重複コード回収できる
  • friend-hearing 関連 docs/ も対象 (md / html)

10GitHub Actions / Code Review 統合

PR を作るたびに 自動コードレビュー + issue triage が走る。

  • portfolio repo は商談用 LP / demo-gallery が頻繁に変わる → セルフレビューを自動化
  • masatoman.net (軸 A) の自動運用と相性が良い (Claude が自動で校正 PR を立てる)
/ultrareview との違い: GitHub Actions は 毎 PR 自動 / 無料枠あり / 軽量、 /ultrareview は 手動 / 課金 / 重い深掘り。

11Plugins で skill / hook / agent / output-style をパッケージ配布

masatoman さんは skill を 14 個以上 自作している → これを plugin にまとめて配布可能 な状態。

  • masatoman-rules-plugin (x-post-rules + masatoman-article-rules + article-publish-gate を 1 セット)
  • 知人開発者に「この plugin 入れれば 同じ運用ができる」 と配布 → 軸 A 認知ルートの 1 つに化ける可能性

推奨着手順 (ROI 軸)

  1. Output Styles の custom 作成 (軸 B 主顧客モード) 1 時間 / 永続効果
  2. fewer-permission-prompts skill 実行 5 分 / 体感差大
  3. /fast モード 試す 即時 / コスト 0
  4. UserPromptSubmit hook で軸 B/A 自動判定 30 分 / STRATEGY.md 遵守を自動化
  5. Web 版 Claude Code (claude.ai/code) を 5/22 面談前に試す 当日のメモ起こしに使えるか確認

ウルトラシンク結論: masatoman さんの workflow に最も効くのは Output Styles の custom 化 と UserPromptSubmit hook で軸切替の自動 guard の 2 つ。 どちらも skill 文化と相性が良く、 既存 memory システムと衝突しない。
