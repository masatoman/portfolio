## 1. 審査員によるクリティーク (20個)

### 池原 真佐子

- **致命的弱点**: 学習能力高いが「伴走投資したくなる人」 に届いていない。 ピボットの論理はあるが、 "誰を幸せにしたいか" より "戦略最適化" が前面。 「6 プロダクト撤退」 を撤退判断の速さとして誇っている点は「また撤退するのでは」 不安に直結。 N=1→3 は営業 KPI、 "事業の解像度がどう変わるか" が見えない。 一次通過はあり得るが「育成したい起業家」 ではなく「器用な個人開発者」 で止まる危険。
- **別案**: 「事業承継インフラ」 は大きすぎる。 "アトツギの右腕" = **翻訳業** に寄せる。 「父親世代の暗黙知を、 息子世代が引き継げる形に翻訳する」。 AI / BPaaS / 承継 / 地域 / 若返り が全部一本につながる。 今の「インフラ」 は金融機関受けを狙いすぎ、 概念が原体験より先に走っている。
- **競合視点**: 他応募者にも「地域 DX」「AI 業務改善」 は大量。 その中で本案だけが持つ強みは "仮説を壊した履歴"。 補助金訴求を捨てた / IT 弱者を捨てた / ROI 誇大表現を捨てた / デモを営業装置に再定義した — ここが異常に良い。 ただし ES に埋もれている。 "変化できる起業家" として構造化し直さないと、 レベルアップ度が伝わらない。
- **極論タイトル採点**:
  - 「多摩発・工務店事業承継インフラ」: 68 点 (大きいが抽象的)
  - 「親方の引退と廃業を切り離す」: 89 点 (情景がある)
  - 「アトツギが継げる工務店を残す」: 92 点 (主語明確)
  - 「70 歳の親方を、 安心して引退させる」: 95 点 (感情がある)
- **背景冒頭**: 「友人が、 夜 21 時に現場写真を整理していた」 は強い。 ただし次の文で「だから私は地域インフラを…」 に飛ぶと死ぬ。 "その姿が、 自分の住む街の未来に見えた" まで情景で繋ぐべき。

### 伊藤 羊一

- **致命的弱点**: 結論が多すぎる。 BPaaS / AI エージェント / 事業承継 / 地域インフラ / アトツギ / 補助金 / デモ — 全部言っている。 結果、 何者かわからない。 本当に言いたいことは「工務店の親方が引退できず、 地域から工務店が消えている」 一つ。 まずこれを置き、 後に なぜ / 誰が困る / なぜ自分が / どう始める を積む。 ES は論文ではない。 審査員は「この人、 走るな」 と思えるかを見ている。
- **別案**: BPaaS → AI の二段ロケットは説明コストが高い。 一次選考では「まず業務代行で入る、 AI は後から作る」 だけで十分。 審査員は AI の構造 / エージェント設計 / アーキテクチャを評価しない。 評価するのは顧客理解 / 入口戦略 / 熱量 / 解像度。 「AI」 は差別化ではなく、 今やノイズになり得る。
- **競合視点**: 他応募者は「AI で効率化」 しか言わない。 あなたは「現状でいい人には売れない」 と言っている。 ここが圧倒的に良い (顧客を捨て / 市場を狭め / 痛みが強い層だけを狙う = 事業家の思考)。 逆に弱いのは「235 件分析」 が多すぎること。 数字は武器にもなるが "調査した感" にもなる。 一次通過する ES は意外と数字を削っている。
- **極論タイトル採点**:
  - 「親方の引退と、 工務店の廃業を切り離す」: 96 点
  - 「アトツギが継げる工務店を、 多摩に残す」: 94 点
  - 「工務店を DX するのではない、 継げる形にする」: 93 点
  - 「夜 21 時の写真整理を終わらせる」: 88 点
  - 「70 歳の親方が引退できない」: 91 点
- **想定顧客頭出し**: 「多摩の工務店では、 "継ぎたい息子" と "変われない父" が分断されている」 = 一発で構造が伝わる

### 佐藤 俊太

- **致命的弱点**: 工数構造が最も危険。 BPaaS は売上が立っても本人稼働依存が極端に高い。 受託 140h / 月 + 新規営業 + ヒアリング + 実装 + 業務代行 + 補助金理解 + デモ更新 を 1 人で回そうとしている。 月商 30 万までは行けても、 その先で本人ボトルネック化の可能性高。 「AI エージェント単価 100-300 万」 は根拠不足。 現状の一次情報では "見積整理に困っている" は確認できても、 "100 万払う" は確認できていない。
- **別案**: 財務的には AI 受託よりも「月 3-5 万円の継続業務を 10 社積む」 (= ¥30-50 万) が遥かに健全。 写真台帳 / 見積整理 / 議事録 / OB 顧客整理 を "事務局代行" として固定収益化した方が、 融資可能性は高い。 現在の構造は BPaaS = キャッシュ / AI = 夢 に分離しすぎ。 金融機関は夢より再現性を見る。
- **競合視点**: 介護 AI BPO が特別賞止まりだったのは「人手不足解決」 で止まったから (利益率 / スケール / 独自資産が弱かった)。 本案も そのままだと同じ場所に落ちる。 ただし違うのは 地域金融 / 承継 / アトツギ / 暗黙知 まで踏み込んでいること。 ここを財務モデルに落とせれば、 単なる BPO から脱出できる。
- **極論タイトル / 頭出し**: 金融機関目線で強いのは「廃業防止 / 承継 / 地域供給維持」、 弱いのは「AI / エージェント / デモ」 (手段に過ぎない)。 頭出し「多摩では、 後継者がいても工務店が継がれない」 が良い (後継者不足には慣れている、 "後継者がいるのに継げない" は新規性)。

### 十河 信介

- **致命的弱点**: 「地域インフラ」 という言葉を使うには地域接続がまだ弱い。 現時点では工務店 1 社 / 北原氏 / たましん / 調査 しかない。 沿線価値を語るなら 空き家 / リフォーム供給 / 職人減少 / 若年定住 への波及まで必要。 今のままだと「工務店向け便利サービス」 に地域語彙を被せている印象。
- **別案**: 「事業承継インフラ」 より「**多摩の住宅メンテナンス供給網を維持する**」 の方が強い。 鉄道 / 不動産 / 沿線人口 / 中古住宅流通 と直結する。 工務店は単独産業ではない、 "街の維持管理レイヤー" として再定義すると 京王沿線との接続が急に本物になる。
- **競合視点**: 他応募者は観光 / 子育て / フード / AI 教育に流れる。 その中で強みは「地域の裏側を支える産業」 を扱っていること。 ただし弱点は **まだ "多摩である必然" が薄い**。 「全国の工務店でも成立するのでは?」 という疑問に答えられないと、 地域コンテストで埋もれる。
- **極論タイトル / 頭出し**:
  - 「多摩の家を守る工務店を、 次世代へ残す」: 94 点
  - 「住宅供給を支える町工務店を、 廃業させない」: 91 点
  - 「親方の引退で、 地域の修繕力が消えている」: 95 点
  - 頭出し: 「多摩では、 家を直せる工務店そのものが減り始めています」 = 沿線価値に直結

### 多摩信用金庫 価値創造事業部 部長

- **致命的弱点**: 率直に「インフラ」 は言い過ぎ。 現状は "高解像度な個人事業"。 一次情報の取り方は非常に良く (補助金幻想を捨て / IT 弱者を捨て / アトツギに寄せた) 修正能力は高く評価できる。 一方で危険なのは「たましんを営業導線として見ている」 ように読める瞬間があること。 金融機関は "紹介しても事故らない人" を見ている。 今の ES は少し「戦略的すぎる」。
- **別案**: 「融資先廃業阻止」 は悪くない、 ただし本当に刺さるのは「**融資先の次世代化**」。 廃業阻止だけでは守り、 アトツギ DX は攻め。 Big Advance との接続も受発注 / 人材 / 事業承継 と自然に繋がる。 "守る" より "次世代へ移す" の方が金融機関は前向きに支援しやすい。
- **競合視点**: 介護 AI BPO が特別賞止まりだったのは 地域接続が弱い / 金融との接点が弱い / 「誰が残るのか」 が曖昧 だったから。 本案はそこを改善している。 ただし まだ N=1 / 実売上ゼロ / PoC なし。 「非常に筋の良い仮説」 止まり。 一次通過は十分あり得るが、 ファイナル級は実績が要る。
- **極論タイトル / 頭出し**:
  - タイトル: 「後継者がいても継げない工務店を、 多摩から減らす」: 95 点
  - 頭出し: 「多摩では今、 "継ぎたい子ども" より先に、 "引き継げる形" が消えています」 = 問題構造が一文で伝わる

## 2. 統合提言: トップ 3 改善

### ① 「AI エージェント」 を半分削る

AI の説明が多すぎて "AI 受託屋" に見える。 本当に強いのは 顧客理解 / アトツギ構造 / 承継課題 / ピボット履歴。 ES では「まず手動で入り、 後から AI 化する」 に圧縮した方が強い。

### ② 「地域インフラ」 を一段下げて "翻訳業" にする

今はスケール語彙が先行。 改善案: 「親方世代の暗黙知を、 アトツギ世代へ引き継げる形に翻訳する」。 これなら AI / BPaaS / 承継 / 地域 / 人 が一本になる。

### ③ 「仮説を捨てた履歴」 を主役化する

本案最大の強みは IT 弱者を捨てた / 補助金幻想を捨てた / ROI 誇張を捨てた / デモの役割を変えた こと。 "市場に合わせて自分の仮説を壊せる" 点が一次通過に直結する。
