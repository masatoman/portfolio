## §1. 3 AI 統合の核となるフレーム

### claude.ai 自身の正直な総括 (最も重要)

> Option A は Round 1「身の丈に寄せろ」 提言の逆方向。 矛盾を解いた瞬間に、 実行可能性という新しい矛盾が生まれた。 ただし Round 2 が示したのは、 **「広げた器 (2 段階・救済・OB 保全・ハブ) は、 ほぼ全て後ろに置ける / 提案に留められる」** ということ。 **「広く見せて、 狭く実行する」 設計が、 矛盾解消と実行可能性を両立させる唯一の道**。

## §5. 対立 (要 masatoman 判断、 統合解あり)

### Q. 多摩信金連携の具体化 vs トーン変更

| | 提案 |
|---|---|
| **Gemini リスクヘッジ #2** | 地域支援グループ等への事前の非公式相談で具体化 |
| **claude.ai 多摩信金** | 許可なく前提に書くのは勇み足、 トーン変更必要 |

**統合解**: 順序として **先にトーン変更 (ES では「提案・お伺い」)** → 受賞後に具体化協議。 ES 提出前に多摩信金との非公式相談はやってもいいが、 結果を ES に書かない方が安全 (実行不確定なため)。 claude.ai 提案が優先。

### R. 入口 N=3 vs N=1 有料 + N=2 試行

| | 提案 |
|---|---|
| Gemini | N=3 を目標として維持 |
| ChatGPT | N=1 有料 BPaaS 1 社目を最重要 |
| claude.ai 池原 | N=3 絶対目標は未達時に全崩壊、 最悪 N=1 有料 + N=2 試行で下限 |

**統合解**: 公開 ES では **「N=3 目標」 (= 成長曲線として書く)** 、 内部では **「N=1 有料 + N=2 試行が下限」** でリスクヘッジ。 10/16 セミファイナル時に下限達成なら成長曲線成立。

### S. アトツギ N=1 vs 父親説得成功 N=1

| | 最重要アクション |
|---|---|
| Gemini | 「アトツギの父親説得成功事例 N=1 を 10 月までに作る + 動画」 |
| ChatGPT | アトツギ N=1 接触 + 手動 BPaaS 1 社目 + 5 分デモ の 3 つ |
| claude.ai | 入口「元気な工務店 + アトツギ」 を 1 社、 有料・手動で回し、 顧客の生のセリフを取る |

**統合解**: 3 段階で全部やる方向:
1. **6 月** = アトツギ N=1 ヒアリング (ES 提出前に「会った」 + 課金意思の言質)
2. **7-8 月** = 手動 BPaaS 1 社目開始 (有料・月 3 万)
3. **10 月** = 父親説得まで踏み込んだ事例化 (動画化が理想だが必達ではない)

→ 6/30 ES 提出時点では「N=1 ヒアリング + 課金意思言質」 だけでも十分。

## §7. ES 設問別 修正アクション (Round 2 反映版)

### 設問 1: ビジネスプラン名

**主案 (Round 2 統合版)**: 「黒字なのに継げずに消える工務店を、 跡継ぎと AI で残す。 — 多摩・工務店承継プロジェクト」 (claude.ai 伊藤「タイトル単体 85 点」 を維持)

**副案**: 「親方の引退と、 会社の廃業を、 切り離す。」

→ 「事業承継インフラ」 単語は副題からも削除候補 (ChatGPT N 採用なら)、 「承継プロジェクト」 で代替

### 設問 2: ビジネスプランの内容 (約 400 字、 単線化)

**Round 2 統合版ドラフト**:
```
本事業は、 多摩のアトツギ世代 (30-40 代、 2 代目 / 3 代目) が継ぐ年商 1-5 億の工務店に対し、 親方世代の暗黙知 (見積もりの癖・顧客対応・現場判断) を AI で次世代に翻訳するサービスである。

入口は業務代行 (BPaaS) で月 3-8 万 / 社、 アトツギが父親世代と並走しながら事業を継げる状態を作る。 蓄積した業務テンプレートは将来的に、 多摩信用金庫様の融資先で後継者問題に悩む工務店にも、 同金庫様のご判断のもと展開を提案させていただきたい。
```

変更:
- 「触媒・ハブ」「地域インフラ」「OB 保全」「翻訳業」 多用 削除
- 出口は「展開を提案させていただきたい」 トーンに (= 統合判定 A + I)
- 入口の「アトツギの自己実現」 を明示 (= Gemini 改善 #2 + claude.ai 池原)

### 設問 3: 想定顧客・課題・ニーズ (約 600 字)

**頭出し**: 「多摩の小規模工務店は、 社長 (60 代・IT 苦手)・経理の妻・跡継ぎ (30-40 代・IT 得意) の 3 人で回っている。 新しい仕組みを本気で欲しがっているのは、 3 人目だけだ。」 (維持)

**重要追加**: アトツギ N=1 ヒアリングのセリフ + 課金意思
- 例: 「実際に話したアトツギは『親父より自分が困っている』 と言い、 月 3 万なら払うと答えた」 ※ 6/30 前に N=1 接触必須

**OB 資産**: **「将来構想として、 廃業時に消失する OB 顧客台帳を、 第三者承継の仕組みで地域内に保全する設計を検討中 (法的スキームは専門家と整備予定)」** まで後退 (= 統合判定 D + O)

### 設問 4: ビジネスプランの特徴・強み (約 400 字、 「仮説を壊した履歴」 主役化)

**Round 2 統合版**:
```
本案最大の強みは、 過去 1 ヶ月で 4 つの仮説を自ら壊してきた事実である:

1. 「IT 弱者 60 代社長」 を主顧客にする仮説 (5/22 北原氏面談で「現状でいい層 = 動機ゼロで売れない」 確認 → アトツギにピボット)
2. 「補助金 1/8 訴求」 を商談武器にする仮説 (制度面 3 段 + 実需面 4 段で崩壊 → IT 通常枠 1/2 のみに圧縮)
3. 「ROI 誇張」 でデモを売る仮説 (誇大広告と判定して数字全削除)
4. 「動くデモ = 営業装置」 という仮説 (「名刺 + 痛みのきっかけ」 に再定義)

これらは全て一次データ (235 件痛みクラスタ + 794 件レビュー + アトツギ N=1 + 北原氏面談) と照合した上での仮説破壊。 市場に合わせて自分の仮説を壊せる起業家としての証拠であり、 4 ヶ月のレベルアッププログラム期間中も同じ強度で学び続ける。

加えて、 建設業特化中小企業診断士 (北原氏) の伴走 + 多摩在住 12 年 + たましん既取引 = 地域ネットワークが手元にある。
```

変更:
- 4 つの強みリスト → 「仮説を壊した履歴 4 つ + 地域ネットワーク 1 つ」 に再構造化 (= 統合判定 G)
- AI エージェント実装力 / 補助金理解 / 撤退 6 件 は削除
- 「触媒・ハブ」 削除 (= 統合判定 E)

## §9. 11/10 までの最重要アクション (3 AI 合意)

**入口の「元気な工務店 + アトツギ」 を、 1 社でいいから "有料で・手動で" 回し、 顧客の生のセリフ (動画 or 引用) を 1 つ取ること。**

5 ペルソナ全員の懸念がこの 1 点に収束:
- 池原: 「絞る力の証明」
- 伊藤: 「N=1 のセリフ」
- 佐藤: 「課金意思 = 財務の背骨」
- 十河: 「実在する施主・現場」
- 多摩信金: 「当金庫を当てにせず独力で動いた事実」

→ N=3 も AI エージェントも出口接続も OB 保全も、 この 1 つの下位互換。 もし時間が割れたら、 **N=3 を捨てて N=1 有料の質に全振り**。
