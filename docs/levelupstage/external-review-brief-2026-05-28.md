工務店プロジェクト (軸 B) の 商談で 真の キーパーソン = アトツギ (2 代目 3 代目、 30-40 代) という insight。

## 結論

工務店 を 「IT 弱者の 社長」 単独で 説得する ルート は 効率が 悪い。 3 層意思決定構造 を 前提に、 アトツギを 窓口化 + 社長を 決裁者 として 並行説得 する のが 正解。

## 3 層意思決定構造

| 層 | 人物 | 年齢 | IT リテラシ | 関心事 | 立場 |
|---|---|---|---|---|---|
| 決裁 | 社長 | 60-70 代 | 弱 (LINE / 電話のみ) | 受注確保・現場・職人手配 | 説得対象 (最終決定者) |
| 経理 | 社長 の 妻 | 60 代 | 中 (帳簿ソフト程度) | 資金繰り・支払い・補助金 | 金銭面で 味方化 |
| 推進 | アトツギ (子・娘) | 30-40 代 | 強 (普通に Web 使える) | 事業承継・効率化・若返り | 真の キーパーソン |

家族経営 が 大半 の 工務店 で、 この 構造 は ほぼ 普遍。 アトツギ は 「父親 を 説得する 材料」 を 欲しがって いる ので、 IT 弱者 の 父親 に 直接当たる より、 アトツギ に SaaS を 渡して 父親 と 経理 の 妻 を 説得 してもらう の が 100 倍 効率 が 良い。

## アトツギ世代の特徴

- IT に 普通に 明るい (LINE 以外 も 使える、 SaaS 比較表 が 読める)
- 「父親 の 現状 を 変えたい」 が 強い モチベ (事業承継 準備)
- SNS は X / Instagram が 多い (Facebook は 低い)
- 「兄ちゃん の SaaS」 ではなく 「業界の 流れ・トレンド・他社事例」 で 説得 されやすい

## 落とし穴

- アトツギ 単独説得 は 無意味: 決裁権 が ない ので、 「いいですね」 と 言わせても 契約 に 至らない。 必ず 父親 同席 の 場 を 作る
- 「父親 を 説得 してください」 と 丸投げ し ない: 説得材料 (補助金 1/2 訴求 / 同業他社事例 / ROI 計算書) を 用意して 渡す
- アトツギ 不在 の 工務店 (一人親方 / 親子 経営でない) は この 構造に 当てはまら ない ので、 別ルート (建材屋 経由) で 攻める
