## P1 (修正済)

### P1-1: demo-gallery 「画像生成待ち」 オーバーレイが実画像を覆っていた

- **場所**: `components/demo-gallery/vote-form.tsx:304-318` (desktop orientation 画像枠)
- **症状**: 5 デモ (estimate / call-memo / site-photo / drawing / website-refresh) で「画像生成待ち (xxx.png)」 のテキスト + svg アイコンが表示。 実画像 PNG は `public/images/demo-mocks/` に全て存在していたが、 絶対配置 placeholder div が手前にあり (透明 PNG 領域では透けて見えた)。
- **主顧客への影響**: 「これ未完成じゃん」 と即離脱 → 投票自体が始まらない。 LINE 拡散の起爆剤を殺す致命傷。
- **修正**: placeholder div + svg を完全削除。 img を `aspect-[16/10]` 枠内に単独配置、 `loading="lazy"` 付き。
- **検証**: モバイル巡回後の `placeholderTexts` 検出で 0 件、 9 枚全 img が `complete: true / naturalWidth > 0`。

### P1-2: demo-gallery OGP / Twitter Card 完全欠落

- **場所**: `app/demo-gallery/page.tsx` metadata
- **症状**: `openGraph` / `twitter` / `canonical` 未設定 → `app/layout.tsx` の portfolio 全体用 OG (「井原誠斗 | 残業の原因を、ITで撃退します。」) が継承。
- **主顧客への影響**: 友人 LINE 拡散時に カード展開で「これ何のページ?」 → 開く前にスルー。 北原氏が URL 共有する場面でも同じ。
- **修正**:
  - `openGraph` (title / description / url / site_name / locale / type=website / images[1200x750])
  - `twitter` (card=summary_large_image / title / description / images)
  - `alternates.canonical = https://ihara-frontend.com/demo-gallery`
  - og:image に `/images/demo-mocks/estimate-organizer.png` (1200x750) 流用
- **検証**: `document.head.querySelectorAll('meta')` で og:title / og:description / og:url / og:site_name / og:image / og:image:width / og:image:height / og:image:alt / og:type / twitter:card / twitter:image 全て確認済。

### P1-3: 画像サイズ 11 MB (主顧客モバイル 4G で 10-20 秒ロード)

- **場所**: `public/images/demo-mocks/*.png` (8 枚)
- **症状**: 2.0 MB / 1.7 MB / 1.4 MB / 1.3 MB / 1.3 MB / 1.1 MB / 1.0 MB / 696 KB の PNG。 8 枚合計 約 11 MB。
- **主顧客への影響**: LINE → モバイル + 4G では 10-20 秒のロード時間。 LCP > 3s で離脱率 ~40% アップ。 完了条件「3 秒以上のローディング」 に直球該当。
- **修正**:
  - `cwebp -q 82 -m 6` で 9 枚を WebP 化 (`voice-daily-report-history.png` 含む)
  - 各画像 24-248 KB に、 9 枚合計 **約 940 KB (92% 削減)**
  - `app/demo-gallery/page.tsx` の `imagePath` を 8 件 `.png` → `.webp` に切り替え
  - `OG_IMAGE` のみ PNG のまま (LINE / Twitter で WebP 非展開リスク回避)
- **検証**: `naturalWidth` 値で全 9 枚デコード成功確認。

### P1-4: `metadataBase` 未設定 (本番で og:image が相対 URL のまま壊れる)

- **場所**: `app/layout.tsx` Metadata
- **症状**: dev 環境では og:image が `http://localhost:3000/images/...` で展開。 `metadataBase` 無しのまま本番デプロイすると og:image の絶対 URL 解決が不安定。
- **主顧客への影響**: 本番 LINE 拡散で OG 画像が出ない可能性 → 主顧客は LINE で OG 画像が出ない URL を踏まない。
- **修正**: `metadataBase: new URL('https://ihara-frontend.com')` を `app/layout.tsx` のメタデータ先頭に追加。 子ページの相対 og:image / twitter:image が本番 URL で自動絶対化される。
