実機能 は どこまで 動く んですか? 録音した内容、 ちゃんと 文字に なります か?
今は UI プロトタイプ 8 種 (画面設計 + 操作フロー) が 完成 した 段階 で、 投票ページで 主顧客 の 反応 を 計測中 です。 実機能 (音声認識・OCR・写真分類 等) は 第 1 号ユーザー の 御社業務 に 合わせて 共同開発 する 設計 です。 御社 で 「日報 で 拾いたい 項目」 「写真整理 の 分類軸」 が 業務 ごとに 違う ので、 UI 完成済 + 御社業務ベース で 実装する のが 一番 ぶれない と 考えました。 IT 導入補助金 + 都 DX 助成金 で 初期開発費 を 1/8 まで 圧縮 する 形 が 自然 です。 第 1 号ユーザー さん には 初期価格優遇 + 機能リクエスト権 + 毎週デモ で 透明性確保 を 特典 として ご提供します。
どんな SaaS なんですか? 既存サービス (ANDPAD 等) と 何が 違うんですか?
IT が 苦手な 60 代 工務店オーナー が 触れる レベルに 振り切った 設計です。 ANDPAD は 多機能で 値段も 月 数万 ですが、 こちらは 「見積整理」 「現場写真整理」 「日報音声入力」 等の 1 機能特化 で 月 1 万円台。 補助金で 実質負担 1/8 まで 圧縮 できる ので、 工務店側の 導入ハードル が 段違いに 低い です。 UI プロトタイプ を 見て いただく のが 一番 早い ので 訪問させて ください。
補助金で 1/8 って、 ほんとに そんな ことできるんですか?
IT 導入補助金 (国、 3/4 補助) + 東京都 DX 推進助成金 (都、 残り部分の 1/2) を 別経費 で 同時申請 する 設計です。 同一経費 への 二重補助 は 不可なので、 SaaS 本体 = 国 / 運用・サーバー = 都 で 分けます。 同時申請 可否の 最終確認 は 5/22 に 東京創業ステーション の 北原氏 (中小企業診断士) に 確認予定 です。 たましん さん の 補助金担当 と 連携 できる と 工務店側 の 採択率も 上がる はず です。

個人事業主 で フロントエンド開発 を 10 年弱 やって おります。 自社サイト ihara-frontend.com で 8 つの 工務店向けデモ を 公開済 で、 業界 SNS / ブログ / Google マップレビュー 計 1,029 件 の 痛み分析 も 一次情報 として 持って います。 訪問時に お見せ します。 取引先 では ない ですが、 たましん さん の 口座 を 長く 使わせて いただいて おり、 多摩地区 に 根を 張って 工務店 さん の DX を 支援したい と 考えて います。

5-30 人規模 の 中小工務店 で、 オーナー さんが 60 代前後 で IT は 苦手だが 「業務改善 したい 気持ちは ある」 という 方 が 一番 刺さります。 2 代目・3 代目 の アトツギ が いて、 父親 を 説得する 材料 を 欲しがって いる パターン も 大歓迎 です。 補助金 を 真面目に 申請 する 工務店 さん は たいてい 当てはまります。
