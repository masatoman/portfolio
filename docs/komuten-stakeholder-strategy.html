CHANNEL STRATEGY

工務店 周辺 ステークホルダー戦略
— 「主顧客 直接」 から 「チャネル 経由」 に 視野 を 広げる

2026-05-11 作成 / masatoman 本人用

視点 転換

主顧客 (一人親方 工務店オーナー) は friend 1 件 しか ヒアリング 接点 が ない。 でも 工務店 周辺 には 設計事務所 / 大工 / 建材屋 / 補助金支援者 / 電気水道 / 信金税理士 という 多様 な ステークホルダー が いる。 彼ら は (1) 自身 が 顧客 に なれる、 (2) 工務店 へ の 紹介経路 に なれる、 (3) ヒアリング 対象 に なれる、 の 3 役。 主顧客 1 件 で 止まらず、 周辺 から 攻める。

01マップ: 工務店 を 取り巻く 6 ステークホルダー

Deep Research と 友人 メモ に 登場 した 主要 ステークホルダー を 「自身 顧客 価値」 と 「工務店 紹介経路 価値」 の 2 軸 で 評価:

ステークホルダー 規模 / 特徴 自身 顧客 価値 工務店 紹介 価値 masatoman の 接触 経路
建材屋 / 卸業者 地域密着・営業マン が 毎日 工務店巡回 中 ◎ 極めて高い LIXIL ショールーム・ナフコ・地元 建材店 巡回
設計事務所 / 建築士 1-3 人、 IT 受容性 中-高 ◎ 高い ◎ 高い (工務店 を 選ぶ 立場) 知人 / 建築士会 / SNS
補助金支援者 / IT 導入支援事業者 IT 受容性 高、 工務店 が 申請 に 来る 中 ◎ 高い 商工会議所 経由 / J-Net21 検索
大工 / 一人親方 主顧客 と 同じ ペルソナ、 IT 弱 ◎ 高い (同じ商品 売れる) 中 (工務店 と は 上下関係) 東京土建 支部 / 知人
電気・水道・ガス業者 大工 と 同じ ペルソナ 中 (大工 と 同じ商品) 中 東京土建 / 各業界団体
信金担当 / 税理士 地域経済 ハブ、 工務店 の 経営 を 知る 低 (SaaS 売れない) 中 (紹介してもらえる 立場) 顔合わせ で 「IT 案件 紹介して」

戦略: ◎ × ◎ が 重なる 建材屋 + 設計事務所 が 主戦場。 自身 にも 売れ、 工務店 への 紹介経路 にも なる。

02各 ステークホルダー の 詳細

★ 主戦場 1

建材屋 / 卸業者 の 営業マン

毎日 〜 週数回 工務店 を 訪問 している プロ。 1 次情報 で 「結局 一番 の 情報源 は 建材屋 の 営業マン が 持って くる 生 の 話」 と 確認 済み。
規模 / 特徴 建材店 (LIXIL 系 / ナフコ / 地元卸 / ロイヤルホームセンター 等)。 営業マン は 1 人 で 数十 工務店 を 担当。 経営者 は 中堅以上 (年商 数億〜)。 IT 受容性 中。
ペイン (推測) FAX 受注 の 電子化 / 顧客カルテ / 取引履歴 管理 / 営業ルート 効率化 / 工務店 ニーズ ヒアリング 集約。
masatoman の SaaS / 受託 候補 ① FAX 受注 OCR + 自動入力 (PDF 化 → みつもろう 17 連携)、 ② 営業 LINE 通話 文字起こし + CRM (どの 工務店 が 何 を 言って た か 記録)、 ③ 工務店 ニーズ レポート (issue-finder 流用 で 業界悲鳴 を 月次 集約)
工務店 紹介 経路 として の 価値 営業マン に 「IT で 困ってる 工務店 いる?」 と 聞ける 関係 を 作れれば 案件 自動流入。 1 次情報 引用: 「建材屋 の 営業マン が 『うち の 息子、 そういう の 得意 なんで』 と 言う から、 小遣い 程度 で HP を 作って もらった」 = この ポジ に なれる。
ヒアリング / 接触 アクション 近所 の LIXIL / ナフコ / 地元建材店 を 月 2-3 回 訪問。 営業マン と 顔つなぎ → 「IT に 詳しい 兄ちゃん」 認知 を 作る。 友人 ヒアリング Q5 で 「親方 は 建材屋 経由 で 業者 を 探す か」 を 検証。
★ 主戦場 2

設計事務所 / 建築士

客 と 工務店 の 間 を 取り持つ ハブ。 友人 メモ で 「客 → 設計事務所 → 工務店」 ルート が 明示。 コンペ で 選ばれる、 長い 付き合い の 工務店 も 持つ。
規模 / 特徴 1-3 人 の 小規模 事務所 が 主流。 建築士 1 級 / 2 級。 IT 受容性 中-高 (CAD / Jw_cad / Revit を 使う)。 工務店 と 違って 「設計 = 知的労働」 を 自認 し、 デジタル 親和性 高い。 年商 1,000 万〜5,000 万。
ペイン (推測) 確認申請 書類 (区役所への 根回し)、 図面 から 見積書 作成、 工務店 マッチング、 客 と の やり取り (LINE で図面送る)、 補助金 申請支援 (省エネ・長期優良住宅 等)。
masatoman の SaaS / 受託 候補 ① 図面 OCR → 自動 見積 (友人 ペイン #1 の 上流 版)、 ② 確認申請 書類 半自動化 (区役所 提出 様式 を テンプレ + 自動入力)、 ③ 客 向け 進捗 共有 アプリ (LINE 経由 で 写真 / 図面 送信、 既読 管理)、 ④ 工務店 マッチング DB (3 社 見積もり を 効率化)
工務店 紹介 経路 として の 価値 設計事務所 が 工務店 を 選ぶ 立場。 仲良く なれば 「IT で 困って る 工務店 いる?」 「補助金 申請 で 一緒 に 動ける IT 屋 紹介 して」 と 言って もらえる。 年 数件 の 高確度 紹介 が 期待 できる。
ヒアリング / 接触 アクション 知人 に 建築士 / 設計事務所 経営者 いない か 確認。 いれば 1 件 ヒアリング (friend と 同じ 20 問 ベース で 業界バリエ)。 いない 場合 は X / Instagram で 「住宅 設計」 タグ の 関東 設計者 アカウント を 探して DM。
高 価値

補助金支援者 / IT 導入支援事業者

IT 導入補助金 の 申請 支援 で 食べている プロ。 工務店 が 補助金 を 取りたい 時 に 自然 と 連携 する 構造。
規模 / 特徴 中小企業 診断士 / 行政書士 / 経営コンサル が 多い。 IT 受容性 高。 主顧客 = 中小企業 全般 (工務店 は その 1 部)。
ペイン (推測) 申請書類 の 二重作業、 顧客 への ヒアリング を 効率化 したい、 採択率 向上 ノウハウ の 共有。
masatoman の SaaS / 受託 候補 ① 申請書類 自動入力 (顧客 の 情報 1 回入力 で 各補助金 様式 を 生成)、 ② 採択率 予測 AI (過去 採択 事例 学習)、 ③ IT 導入支援事業者 登録 後 の 連携 (memory: subsidies-positioning.md と 整合、 マイクロ法人化 後 に 棚上げ で OK)
工務店 紹介 経路 として の 価値 ★★ 補助金 申請 が 必要 な 工務店 は 「IT 導入」 の 動機 を 既に 持って いる = 売りやすい セグメント。 支援事業者 と 連携 すれば 工務店 オーナー が 自然 と masatoman に たどり着く 構造。
ヒアリング / 接触 アクション 府中・立川 商工会議所 で 「補助金 申請 を 手伝って くれる 事業者」 を 紹介 して もらう。 1 件 ヒアリング。 ただし memory: business-form-roadmap.md より、 IT 導入支援事業者 登録 は マイクロ法人化 2026/6 後 に 棚上げ なので、 「連携 のみ」 で 進める。
中 価値

大工 / 一人親方 (主顧客 と 同 ペルソナ)

主顧客 (工務店オーナー) と ペルソナ が ほぼ 同じ (IT 弱、 一人親方、 年商 数百万〜1500万)。 同 商品 で 売れる 可能性 大。
規模 / 特徴 友人 メモ より: 大工 は 工務店 と 上下関係 (ほぼ言いなり)。 専門分野 (床だけ / 内装 / 全部できる) で 分かれる。 元請け 監督 と LINE でつながる。
ペイン (推測) 確定申告 / インボイス 対応 (memory: 全建総連 アンケート で 15-16% が 消費税分 受け取れて ない)、 写真台帳 (元請けから 求められる)、 単価 交渉 / 日当 計算 (1.4-2.5 万 が 相場)、 労災保険 加入率 42.3% で 半数 が 未加入。
masatoman の SaaS / 受託 候補 ① 確定申告 + インボイス 自動化、 ② 写真自動仕分け (issue-finder 動くべき度 71 の 真のイシュー)、 ③ 日当 計算 + 給与 シミュレータ
工務店 紹介 経路 として の 価値 大工 自身 が 工務店 を 直接 紹介 する ことは 少ない (上下関係 ある ため)。 ただし 「親方 つながり」 で 横展開 が 可能。
ヒアリング / 接触 アクション 東京土建 多摩西部 支部 訪問 (komuten-channel-strategy.md ToDo と 共通)。 住宅デー (6/25) で 直接 接触 する 機会 も あり。
中 価値

電気・水道・ガス 業者 (職人)

大工 と ほぼ 同じ ペルソナ。 同じ 商品 で 売れる が、 業界団体 が 別 (電気工事工業組合 等)。
masatoman の SaaS / 受託 候補 大工 と 同じ (確定申告 / インボイス / 写真台帳 / 日当 計算)。 業界別 に カスタマイズ する より、 「一人親方 共通 SaaS」 として 横展開 する のが 効率 良い。
ヒアリング / 接触 アクション まず 大工 で 動作確認 した あと、 横展開 で 触る。 ヒアリング は 後回し。
低 価値 (連携のみ)

信金担当 / 税理士

SaaS を 直接 売る 相手 ではない。 だが 工務店 の 経営 を 知って いて、 IT 案件 を 紹介 して くれる 立場。
工務店 紹介 経路 として の 価値 ★ 顔合わせ + 「IT で 困って る 顧客 いたら 紹介 して」 と 仕込み。 「補助金 申請 で IT 屋 が 必要 な ケース」 で 思い出して もらえる ポジ を 作る。
ヒアリング / 接触 アクション 既に komuten-channel-strategy.md の ToDo に 「四半期 1 回 顔合わせ」 で 記載 済み。 ヒアリング 対象 ではない。

03友人 1 件 だけ じゃ ない、 ヒアリング 拡張 案

masatoman は friend (現場監督) しか 工務店 周辺 の 接点 が ない 状況。 でも 周辺 ステークホルダー で 追加 ヒアリング 候補 を 作れる:

対象 探し方 ヒアリング 内容 優先度
建材屋 営業マン 近所 の LIXIL / ナフコ / 地元建材店 訪問 (アポなし で 営業マン と 雑談) 工務店 が IT で 困って る こと、 「自分 が 何 か 手伝える か?」 ★ 最優先
設計事務所 経営者 知人 / X / Instagram「住宅 設計」 関東 タグ / 建築士会 多摩支部 工務店 マッチング、 確認申請、 図面 → 見積 の 業務 ペイン ★ 最優先
補助金支援者 府中・立川 商工会議所 経由 で 紹介 申請書類 の ペイン、 連携 可能性、 IT 導入支援事業者 登録 メリット 高
大工 (友人 の 知人) friend に 「親方 で 話 が 聞ける 人 いない?」 と 依頼 確定申告・インボイス・写真台帳 の ペイン 高
電気・水道・ガス業者 大工 ヒアリング 後 に 横展開 業界差 が ある か の 確認 中
注意: friend ヒアリング (現場監督) は 1 件 で 終わらせない。 そこ で 「大工 知人 紹介 して」 「設計事務所 で 仲 が いい 所 ない?」 と 1 つ ずつ 派生 さ せる。 1 件 → 3 件 → 10 件 と 雪だるま 式 に 広がる の が 紹介経済 の 構造 (Deep Research 引用 [55][56] で 確認 済み)。

04優先 アクション (3 ヶ月 で やる こと)

アクション 狙い 期限
★ 近所 の 建材屋 営業マン と 顔つなぎ (LIXIL / ナフコ 等 月 2-3 回) 「IT 兄ちゃん」 ポジ を 作る → 工務店 紹介 経路 確保 2026/5 月 中 開始
★ 設計事務所 経営者 1 件 ヒアリング (知人 / SNS で 探す) 客 → 設計事務所 → 工務店 の 上流 を 押さえる 2026/6 月 中
friend ヒアリング 後 に 「大工 知人」 を 1 件 紹介 して もらう 大工 ペルソナ の 確定申告・写真台帳 ペイン 検証 friend ヒアリング 後 1 ヶ月 以内
商工会議所 窓口 で 補助金支援者 を 紹介 して もらう 連携 ルート の 入口 作り (受注 案件 の 紹介 を 期待) 2026/6-7 月
住宅デー (6/25) に 関われたら、 建設組合 経由 で 親方 連 と 一気 に 顔合わせ 主顧客 接触 を 1 件 → 複数件 に 拡張 2026/6/25 当日
電気・水道・ガス 業者 は 大工 検証 後 に 横展開 「一人親方 共通 SaaS」 として 売れる か 確認 2026/8-9 月

05戦略 的 含意

含意 1: SaaS 商品 ラインナップ を 「主顧客 (工務店オーナー) 向け」 だけ で なく、 「設計事務所向け」 「建材屋向け」 「補助金支援者向け」 も 視野 に 入れる。 既存 5 デモ (memory: saas-pricing.md) は 工務店 向け だが、 設計事務所 / 建材屋 向け に 横展開 できる 機能 が 多い (図面 OCR / 顧客 CRM / 補助金書類)。
含意 2: 「工務店 オーナー 直接 営業」 は 0 で OK。 建材屋 営業マン と 設計事務所 を 押さえれば 自然 に 紹介 が 来る 構造。 Deep Research の 「紹介 51.5%」 は この 構造 の 数値 化。
含意 3: 主顧客 一本足 思考 から 抜ける。 ヒアリング 対象 を 1 → 5 件 に 広げれば、 ペルソナ の 解像度 が 一気 に 上がる。 5 件 と は: friend (現場監督) + 建材屋 営業マン + 設計事務所 + 大工 + 補助金支援者。 これ で 工務店 を 取り巻く 経路 が 立体 視 できる。
