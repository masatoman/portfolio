## 0. 結論 (3秒で読む)

masatoman さんが「個人事業主」のまま IT 導入支援事業者になるには、 **既存のコンソーシアムに「構成員」として 加わる** 一択。 法人 (単独) では 申請できない。 構成員として加わるためには、 「ITツールを 中小企業に売った 販売実績 (1社以上)」 が 必要。 LaunchKit を中小企業向けに 再ポジショニングして 1社販売実績を作るのが 現実的な動き出し。

---

## 1. 個人事業主の最大の制約

> **個人事業主は「法人(単独)」での登録はできません。**
> (マニュアル P.9)

masatoman さんが取れる選択肢は3つだけ:

| 選択肢 | 必要なもの | コスト感 |
|---|---|---|
| **A. コンソーシアム構成員として既存幹事社に加わる** | 販売実績1社以上 + 幹事社探し + 構成員契約 | 低 |
| **B. 法人化して「法人(単独)」 で登録** | 法人設立 + 1期決算待ち | 中〜高 |
| **C. 自分が幹事社になってコンソーシアム形成** | 構成員1者以上を集める + 自分が法人 | 高 |

A が現実的。 B は 「1期の決算」 が要件 (新設法人だと 1年待ち)。

---

## 2. もう一つの致命的制約: 重複登録不可

> 「IT導入支援事業者(構成員を含む。)」として登録されている、または登録申請中の事業者は、「補助事業者」として交付申請を行うことはできません。 (マニュアル P.2)

つまり登録すると、

- ❌ masatoman さん自身が **業務改善助成金 / 持続化補助金 / 自治体IT補助金で 機材導入** はできなくなる
- ❌ masatoman さんが 別事業者として 補助金を受けることもできない (代表者の他事業者経由も禁止)

**判断: masatoman さんが 自分の事業で補助金を受けたい場合は、 IT導入支援事業者登録 する前に 受けておく**。 一度 登録すると 受給権利を失う。

---

## 3. 個人事業主が「構成員」として登録する場合の必要書類 (5点)

(マニュアル P.14)

| # | 書類 | 注意点 |
|---|---|---|
| ① | 運転免許証 / 運転経歴証明書 / 住民票の写し | 運転免許証は有効期限内、 住民票は3ヶ月以内、 マイナンバーは黒塗り |
| ② | 所得税の納税証明書 (その1 or その2) | 税務署発行、 直近分、 PDF形式の電子納税証明書も可 (XMLは不可) |
| ③ | 確定申告書の控え | 令和7年分が原則 (やむを得ない場合は令和6年分も可)、 税務署受領印 or 受信通知が必要 |
| ④ | 所得税の青色申告決算書 又は 収支内訳書 | 青色なら青色申告決算書、 白色なら収支内訳書、 直近分 |
| ⑤ | **販売実績一覧** | 事務局指定 Excel 様式、 ITツールごとに最低1社の取引先情報 |

⚠️ ① ② ③ ④ ⑤ は **コンソーシアム幹事社からは閲覧不可** (個人情報保護のため、 直接事務局のみ閲覧)。 安心して提出可。

## 4. 「販売実績一覧」 の中身 (これが最大の壁)

(マニュアル P.17)

各 ITツール について以下を記載:

- ITツール名
- 取引先名 / 組織形態 / 法人番号 / 代表者氏名 / 代表電話番号 / 所在地

**ITツール 1種類につき、 少なくとも 1社の販売実績** が必要。 事務局から **契約書 / 請求書 の追加提出** を求められる場合もある。

つまり「ITツールを実際に 中小企業に売って お金をもらった証拠」 が 物理的に必要。 受託開発で 顧客のために組んだ単発のシステムは 「ITツール」 としてカウントされない。

---

## 5. 「ITツール」 として登録できるもの (まだ完全には不明)

このマニュアルでは触れられていないが、 別途「ITツール登録要領」が 存在する。 一般的な要件としては:

- ✅ 既製の SaaS / パッケージ / オンラインサービス
- ✅ 中小企業の生産性向上に直接資するもの (会計、 業務管理、 CRM、 在庫管理、 勤怠 等)
- ❌ 顧客ごとにスクラッチで組む受託開発
- ❌ ホームページ制作 単体 (通常枠では通常 対象外)
- ❌ コンサルティング 単体

masatoman さんの **LaunchKit** は SaaS スターターキットで開発者向け。 中小企業向け業務生産性ツールとして再ポジショニングする必要あり。

## 6. 事業者情報の入力で 書く内容 (P.32〜P.39)

申請書には以下を 詳細に書く必要:

### 基本情報
- 屋号 / 現住所 / 生年月日 / 業種コード / 営業所所在地 / 担当者連絡先

### 財務情報
- 決算月 / 売上高 / 経常利益 / 借入金 / 従業員数

### 自社製品・サービス (P.35)
- 代表的なITツールの概要紹介
- 導入事例: **どのようなユーザーの課題解決に繋がったか**、 具体的な例 (導入先業種、 従業員規模、 導入数量、 導入時期 等)

### 本事業への取組み (P.36) — 事業計画書相当
1. 社内において どのように本事業を周知するか
2. 中小企業に対して どのように本事業を周知し、 利用を促すか
3. IT導入支援事業者として、 生産性向上を目的とする本事業に どのように取り組むか

### ハードウェア販売予定の有無
- 「販売予定なし」 で登録すると 後から ハードウェア販売はできない
- 必要なら 最初から 「販売予定あり」 に

### セキュリティ・認証
- ISMS / プライバシーマーク / SECURITY ACTION 等の取得状況
- 認定証の添付は不要 (申告のみ)

## 7. 申請の流れ (8ステップ、 構成員パターン)

```
[幹事社の作業]
  1. 幹事社 が IT 事業者ポータル で 仮登録 → 事業者情報入力
  2. 幹事社 が 構成員ポータル の アカウントを 発行 (masatoman 用)

[masatoman の作業]
  3. 構成員ポータルにログイン → 必要書類添付 + 情報入力
  4. 「入力完了」 ボタンで 幹事社に 承認依頼

[幹事社の作業]
  5. 幹事社 が 構成員情報を 確認 → 「承認」
  6. 幹事社 が ITツール情報を入力 → 事務局へ提出

[事務局]
  7. 事業者・構成員・ITツール それぞれ別々に審査
  8. 全部の審査が完了したら 登録可否を決定
```

ポイント: masatoman さん の作業は 主に Step 3〜4。 幹事社が大半を進める。 だから **信頼できる幹事社を見つける** のが最重要。

## 8. 不採択になるケース・落とし穴

- **販売実績ゼロ** → そもそも書けない
- **「ITツール」 として 受託開発を 申請** → カウント外
- **不備訂正期日に間に合わない** → 不採択 (同一年度内 再申請不可)
- **本事業への取組み記述が薄い** → 「やる気がない」 と見られる
- **事業計画と販売実績の乖離** → 矛盾と見られる
- **添付書類の有効期限切れ** (履歴事項全部証明書 / 住民票 = 3ヶ月以内、 など)

## 10. 注意: 棚上げ判断の選択肢

masatoman さんが 以下の状況なら、 **IT導入支援事業者にならない** 方が良いかも:

- 自分の事業で 業務改善助成金 / 持続化補助金 で 機材導入したい場合
  - → 重複登録不可なので、 補助金受給を捨てることになる
- LaunchKit を 中小企業向けに 再設計する 余力がない
  - → 販売実績作りで 詰まる
- 申請書に書く時間が 受託の妨げになる
  - → 受注機会損失が大きい

これらに該当する場合は、 **「他補助金 (持続化 / 業務改善 / 自治体) への申請支援」 で 顧客を支える** 路線を 維持し、 IT導入補助金は 諦める。

## 補足: AI ポジショニングとの関係

メインのポートフォリオでは AI を表に出さない方針 (`memory/ai-positioning.md`)。 一方で 制度名が「デジタル化・AI導入補助金」 に変わった 2026 年は、 申請書 (本事業への取組み記述) では **AI 強みを 押す** 方が 通りやすい可能性。

- 顧客向け表面: AI を出さない (社長層は流行り物に距離を置く)
- 申請書面: AI を押す (制度名に 「AI」 入った = 審査員は AI 関連を 評価しやすい)

訴求対象が違うだけで、 矛盾ではない。
