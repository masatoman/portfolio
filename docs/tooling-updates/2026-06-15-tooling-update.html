軸 B 適用 サブエージェントが最大5階層でサブエージェントを生成可能に v2.1.172 / 2026-06-10

Agent ツール呼び出しのネストが最大5レベルまで可能になった。 これまでサブエージェントはさらに別エージェントを起動できなかったが、今後は「ルーティン → 検索エージェント → 評価エージェント → 生成エージェント」のような多段チェーンを1コマンドで走らせられる。 issue-finder v6 のマルチプロファイル処理 (建設業 / 電設 / 設備設計 等) を並列サブエージェントに割り当てるアーキテクチャが現実的になった。

推奨アクション: issue-finder v6 daily queue の設計を見直し、プロファイルごとのサブエージェント並列実行が可能か scripts/ 配下のスクリプトで確認する。 具体的には「親エージェントが6経路ヒアリング候補を並列評価する」設計への応用を検討。 工数目安: 設計レビュー30分。実装は issue-finder v7 に向けたバックログ化を推奨。
軸 B 適用 /cd コマンド — プロンプトキャッシュを保持したままディレクトリ移動 v2.1.169 / 2026-06-08

新コマンド /cd <path> でプロンプトキャッシュを破棄せずに作業ディレクトリを変更できるようになった。 従来の cd はキャッシュをリセットしていたため、daily-operator が portfolio repo から参照ファイルを読み込んだ後に masatoman.net repo へ移動すると再読み込みが発生していた。 7 ルーティン中で複数 repo を跨ぐケースで速度が向上する。

推奨アクション: daily-operator / night-writer 等のルーティン冒頭で cd している箇所を探し、 /cd に置き換えられるか確認する。 特に portfolio memory を読んだ後に masatoman.net へ移動する flow が対象。 工数目安: ルーティンスクリプト確認 20 分 + 置き換え 5 分/箇所。
軸 B 適用 --safe-mode フラグ — 全カスタマイズを無効化してトラブルシューティング v2.1.169 / 2026-06-08

claude --safe-mode で CLAUDE.md / hooks / MCP / settings / skills を全て無効にした状態で起動できる。 既存 hooks が 5 種 (PreToolUse / Edit-Write / Stop / PreCompact / UserPromptSubmit) あり、 ルーティンが増えるにつれて「どの hook が干渉しているか」の切り分けが困難になっていた。 AXIS-GUARD 誤発火や hook 循環など、原因不明のエラー発生時に最初に試すべきコマンドになる。

推奨アクション: CLAUDE.md のトラブルシューティングセクション (または memory の hook 関連ファイル) に 「原因不明のエラー → まず claude --safe-mode で動作確認」を1行追記する。 工数目安: 5 分。
軸 B 適用 language 設定でセッションタイトルを日本語固定 v2.1.176 / 2026-06-12

セッションタイトルが会話の言語を自動検出して生成されるようになった。 さらに settings.json に "language": "ja" を設定することで日本語を強制固定できる。 現状、英語のファイル名が混在している Obsidian Vault の reports/ (616 件) への新規追加分が 日本語タイトルに統一される。

推奨アクション: .claude/settings.json に "language": "ja" を追加する (工数5分)。 既存616件のファイル名は変わらないが、今後追加される reports が日本語タイトルで整理され、 Obsidian の日本語バックリンク検索と相性が良くなる。
