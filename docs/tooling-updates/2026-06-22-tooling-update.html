主軸適用 ⚠️ routine 確認必須 Claude Code v2.1.183 — 自動モード (auto) で破壊的 git コマンドをブロック

2026-06-19 リリース。git reset --hard / git checkout -- . / git clean -fd / git stash drop 等が auto モードで実行不可になった。 また git commit --amend はセッション中に作成していないコミットへの適用もブロック。 terraform destroy / pulumi destroy / cdk destroy も明示的な依頼がない限りブロック。

⚠️ 要確認: daily-operator / content-executor / issue-finder 等の routine が自動実行で git reset / checkout 等を呼んでいないか確認。 routine の CLAUDE.md に「このコマンドを明示的に許可する」旨を書くか、コマンド自体を safer な代替に変更する必要がある可能性あり。 git push / git add / git commit は影響なし。
推奨アクション: 既存 routine の Bash コマンドをざっと grep して reset --hard / checkout -- / clean -f が含まれていないか確認 (grep -r "reset --hard\|checkout --\|clean -f" ~/.claude/)。 該当があれば routine の説明文に「明示的に実行していい」旨を追記するか、コマンドを変更する。 記事ネタ: 「Claude Code auto モードの安全柵が強化された週 — routine 設計で気をつけること」として実装ログ記事に昇格できる題材。
主軸適用 routine 設計 Claude Code v2.1.181 — /config key=value でプロンプト内から設定変更が可能に

2026-06-17 リリース。例: /config thinking=false と書くだけでセッション内の思考モードを即座に切り替えられる。 これまで CLAUDE.md やコマンドライン引数でしか設定できなかった項目をプロンプト行で動的に変更できる。

推奨アクション: routine の先頭に /config thinking=false 等を差し込むことで、routine ごとに思考モードや model を切り替えるパターンが実現できる。 weekly-content-planner (重い) と daily-operator (軽い) で設定を分けるユースケースに直結。 記事ネタ: 「/config コマンドで routine ごとに Claude の思考深度を切り替える設計」。
主軸適用 ⚠️ モデル確認 Anthropic — Claude Sonnet 4 / Opus 4 が 2026-06-15 に退役完了

2026-06-15 に Claude Sonnet 4 と Claude Opus 4 の退役期限が到来し、API 上で利用不可になった。 同時に v2.1.183 で「非推奨モデルへの警告機能」が追加されており、古いモデルを routine で指定しているとアラートが出るようになった。

⚠️ 要確認: CLAUDE.md / settings.json / routine の prompt に claude-sonnet-4 / claude-opus-4 が直接指定されていないか確認。 指定がある場合は claude-sonnet-4-6 / claude-opus-4-8 等へ移行が必要。
推奨アクション: grep -r "claude-sonnet-4[^-]\|claude-opus-4[^-]" ~/.claude/ /home/user/portfolio/ で旧モデル指定を検索。ヒットしたら最新モデル ID に更新。 記事ネタ: 「モデル退役サイクルを routine に組み込む — 自動検知と移行フローの設計」。
