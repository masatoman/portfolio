軸 B 工務店プロジェクトに使える更新

軸 B 適用 Claude Code v2.1.157 — .claude/skills 自動読み込み + claude plugin init + スラッシュコマンド補完 2026-05-29

.claude/skills ディレクトリに置いたプラグインが Claude Code 起動時に自動読み込みされるようになった。 claude plugin init <name> で skill のスキャフォールディングが可能になり、スラッシュコマンドの自動補完も追加された。 ワーカートリー管理も改善されており、並列エージェント作業の安定性が向上している。

推奨アクション: issue-finder v6 のルーティンを .claude/skills/issue-finder/ として切り出し、 claude plugin init issue-finder でテンプレートを生成して整理する。 demo-gallery の投票状況チェックも /demo-gallery-check コマンドとして skill 定義しておくと、 毎週月曜の tooling-update-watch 起動時にスラッシュコマンドとして呼び出せる。 既存の 7 routines (night-writer, daily-operator 等) との競合が出ないか、 /reload-skills で即時テストすること。
軸 B 適用 Anthropic / Claude Code Claude Opus 4.8 + Dynamic Workflows — 最大 1,000 サブエージェント並列オーケストレーション 2026-05-28

Opus 4.8 (v2.1.154, May 28) と同時に Dynamic Workflows が Claude Code に搭載された。 Claude が JavaScript オーケストレーションスクリプトを自動生成し、最大 1,000 のサブエージェントを並列起動できる。 中断しても再開可能 (resumable runs)。MAX/Team/Enterprise プラン対象の research preview。 価格は Opus 4.7 と同じ ($5/$25 per million tokens)。Fast mode は以前より 3 倍安くなった。

推奨アクション (今すぐではなく、9月以降の主軸確定後に検討): プラン C の 6 経路並列ヒアリング (北原氏 / たましん / 建材屋 / 設計事務所 / 多摩商工会議所 / アトツギ甲子園) の結果データが溜まった時点で、Dynamic Workflows を使った「候補 × シグナル」マトリクスの自動更新パイプラインを検討。 現時点では research preview なので本番 workflow には組み込まず、まず demo-gallery の集計スクリプトで動作確認するのが安全。
今週すぐに試すなら: claude CLI で "demo-gallery の 8 デモを並列で評価して需要スコアを算出してください" という指示で Dynamic Workflows が起動するか確認する。
軸 B 適用 Claude Code v2.1.152 — /code-review --fix 改善 + disallowed-tools + /reload-skills + GFM タスクリスト 2026-05-27

/code-review --fix がレビュー結果を自動適用できるよう改善された。 skill / スラッシュコマンドに disallowed-tools を設定できるようになり、 特定 skill が触れるツールを制限できる。 /reload-skills で skill を再読み込み可能。 マークダウン表示が GitHub Flavored Markdown のタスクリスト (- [ ]) に対応。

推奨アクション: local-business/ や demo-gallery/ の TSX 変更後に /code-review --fix を定常ルーティンとして組み込む。 tooling-update-watch の PreToolUse hook に disallowed-tools: ["Edit", "Write"] を設定すれば、情報収集専用 skill が誤ってファイルを書き換えることを防げる。

Obsidian Vault 育成に使える更新

Obsidian 適用 Obsidian Obsidian v1.13.0 — Obsidian CLI 導入 + 設定パネル刷新 + 画像ドラッグリサイズ 2026-05-28 (WebSearch 取得)

Obsidian CLI が導入され、ターミナルから Obsidian を制御・スクリプト化できるようになった。 設定パネルが独立ウィンドウで開くよう刷新され、設定名・説明でのキーワード検索とキーボードナビゲーションが追加。 Live Preview でのドラッグによる画像リサイズ (コーナーをドラッグ、ダブルクリックでリセット) も追加された。 ファイル削除時に添付ファイルも同時削除するか確認するプロンプトが追加された。

推奨アクション: obsidian-cli を使って、tooling-update-watch の Step 5 (commit + push) 完了後に 生成した HTML を自動的に ~/.claude/projects/-Users-master-home-work-portfolio/reports/ へコピーするワンライナーを Stop hook に追加する。 現状は「手動でコピー」と注記しているが、CLI 化で毎週の手動作業がゼロになる。 まず obsidian-cli --help で利用可能コマンドを確認すること (CLI の API は公式ドキュメント要確認)。
Obsidian 適用 Claude Code v2.1.153 — MCP サーバー接続性改善 + バックグラウンドセッション安定性向上 2026-05-28

MCP サーバーへの接続安定性が改善され、バックグラウンドセッションでのエージェント管理が強化された。 具体的な失敗パターンが修正されており、長時間の routine 実行中に MCP ツールが切断されにくくなる。

推奨アクション: pdca-weekly や night-writer など、長時間バックグラウンド実行する routine での MCP ツール呼び出し失敗が減っているはず。 Vault の memory ファイルを MCP 経由で操作するツールがあれば、 今週の routine 実行ログで失敗率の変化を確認しておく。
