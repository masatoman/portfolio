軸 B 工務店プロジェクトに使える更新

軸 B 適用 Claude Code v2.1.142 — 2026-05-14 claude agents コマンドに --model / --effort フラグ追加

claude agents コマンドに --model、--effort、--permission-mode、--settings、--mcp-config、--add-dir などのフラグが追加された。 サブエージェント単位でモデルとコスト水準を明示的に制御できるようになった。

推奨アクション: issue-finder v6 のマルチプロファイル実行で、軽量タスク（ラベル分類）は --model claude-haiku-4-5、重要判定は --model claude-sonnet-4-6 と明示指定することで、1 日のクレジット消費を抑えながら精度を維持できる。 --permission-mode acceptEdits を daily queue に付与して確認ダイアログを省略する運用も検証価値あり。
軸 B 適用 Claude Code v2.1.144 — 2026-05-19 バックグラウンドセッションの /resume 対応・経過時間表示

claude --bg で起動したバックグラウンドセッションを /resume コマンドで再接続できるようになった。 完了通知に経過時間も表示されるため、長時間タスクの進捗把握が容易になった。

推奨アクション: demo-gallery の 8 デモ改修を夜間バックグラウンドで走らせながら、翌朝 /resume で結果確認する運用が実用になった。 特に friend hearing データ整理や issue-finder daily queue の自動実行後レビューで、途中経過を失わずに再接続できる。
軸 B 適用 Claude Code v2.1.143 — 2026-05-15 ストップフックの無限ループ修正

Stop フックが条件次第で無限ループに陥るバグが修正された。 masatoman 環境は PreToolUse / Edit-Write / Stop / PreCompact / UserPromptSubmit の 5 種フックを運用しており、Stop フックの不安定が daily-operator や night-writer の異常終了につながっていた可能性がある。

推奨アクション: v2.1.143 以降に更新済みであることを claude --version で確認。 Stop フック起因と思われる過去の異常終了ログ（~/.claude/projects/*/reports/ 内）を 1 件だけ抽出して再発していないか照合する。
軸 B 適用 Anthropic News 発表日: 2026-05-18 Anthropic が Stainless を買収（SDK 品質向上・MCP サーバーツーリング強化）

Stainless は全公式 Anthropic SDK の生成を担ってきた企業で、MCP サーバーツーリングにも強み。 買収により Anthropic SDK の安定性・型安全性・バージョン追従速度が向上する見込み。

推奨アクション: 軸 B SaaS の将来的な Anthropic SDK 利用時（見積書・工程管理への AI 機能追加）に向け、今後リリースされる SDK メジャーバージョンアップのリリースノートを月 1 回確認するサイクルを設ける。 現時点では既存コードへの変更不要。
