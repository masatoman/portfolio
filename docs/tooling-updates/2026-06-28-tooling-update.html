AI エージェントにセッションをまたいだ長期記憶を付与するオープンソースのメモリプラットフォーム。 Claude Code 専用プラグインが提供されており、プロンプト・ツールトレース・応答内容をセッションメモリに記録し、次のプロンプト投入時に関連コンテキストを自動インジェクションする。 セッション終了時に永続ストレージ（Postgres/Neo4j/SQLite）へ同期する設計。Apache-2.0 ライセンス。

バックエンドは Postgres（デフォルト）、Neo4j、SQLite、Redis など複数から選択できる。 Python パッケージ・TypeScript・Rust クライアント・Docker イメージが揃っており、Cognee Cloud（マネージド版）も利用可。

効率厨の一言: CLAUDE.md + memory/ ディレクトリで手動管理している「セッション間のコンテキスト引き継ぎ」を自動化できる可能性がある。 ただし Postgres などの DB インフラが必要で「今日すぐ」ではない。 routine が増えて memory ファイル管理が限界に感じてきたタイミングで評価するのが正しい順序。 プラグイン経由なのでコードへの干渉が少ない点は評価できる。
