# デザイン改善ヒアリングシート (テンプレート)

> 使い方: ファイルをコピー (例: `design-brief-2026-05-10-top.md`) → 各セクションを埋めて Claude (Code or .ai) に渡す。
> 全部埋めなくても OK。 最低限 ★ マークの項目だけは答える。

---

## 1. 改善対象 ★

- **対象 URL / 画面名**: 例) `/` トップページ、 `/portfolio` ヒーローのみ、 デモカードのレイアウト
- **改善範囲の単位**: ページ全体 / 特定セクション / 個別コンポーネント / コピー文言だけ
- **現状のスクショ or キャプチャ**: あれば添付 (なければ「`/`」のように URL でも可)

## 2. ターゲットペルソナ ★

> 「誰が見て」「何を判断する」 ページか。 抽象論ではなく具体的な 1 人を思い浮かべる。

- **役職 / 立場**: 例) 工務店の現場監督 (45 歳、 男性、 IT は LINE と Excel ぐらい)
- **流入経路**: 例) 商工会議所からの紹介、 Google 検索 「工務店 ホームページ 改修」、 名刺 QR
- **見た瞬間の心理**: 例) 「Web 屋 多すぎてどこも同じに見える」「自分には難しすぎないか」
- **このページで判断したいこと**: 例) 「この人に頼めそうか / 自分の仕事に分かってくれそうか」

## 3. ゴール / 行動誘導 ★

- **このページで取らせたい 1 アクション**: 例) 問い合わせフォーム送信、 LINE 追加、 デモを 1 つ見る
- **その次のアクション**: 例) Google Meet 30 分予約、 LINE で写真送ってもらう
- **離脱しても OK な人**: 例) 法人本社の Web 担当者 (=ペルソナ外)

## 4. 訴求軸 (3〜5 個)

> 「何が刺さる」 か。 競合より優位な点。

- 軸 1: 例) 全部一人で完結 (営業 / 制作 / 業務改善 / 公開後の保守)
- 軸 2: 例) 補助金で半額〜2/3 補助になる可能性を一緒に確認できる
- 軸 3: 例) 月 5,000 円以下で動く構成 = ランニングコストが安い

## 5. 避けたい印象 / NG ワード

- 避けたい印象: 例) 「派手」「ゲーム会社っぽい」「IT に詳しくない人を置き去りにしている」
- NG ワード: 例) 「DX」「アジャイル」「スプリント」「モバイルデバイス」「SaaS」
- 触れない方が良い競合: 例) 「他の Web 屋が高すぎる」 を直接言わない

## 7. トーン & ムード

- **言葉のトーン**: 丁寧 / カジュアル / 技術的 / 業界用語控えめ — どれ?
- **色のムード**: 暖色 / 寒色 / 高彩度 / 低彩度 / モノクロ寄り
- **書体の方向**: ゴシック (現代的) / 明朝 (落ち着き) / 手書き / モノスペース (テック)
- **写真 / イラスト**: 写真メイン / イラストメイン / 抽象的グラフィック / 文字主体

## 8. 制約

- **技術制約**: 例) Next.js 15 + Tailwind 3、 既存 `LabToolPageShell` を踏襲、 SSG 限定
- **ブランド制約**: 例) ロゴ「Masato Works」、 メインカラー `#7a5c38`、 既存フォント Noto Serif JP
- **法令 / 規約**: 例) 補助金訴求は「採択保証なし」を必ず明記、 個人情報フォームは利用規約リンク必須

## 9. 成功の基準

- **数値で測るもの**: 例) `/#contact` への到達率、 LINE 追加率、 デモページの直帰率
- **感覚で測るもの**: 例) 「うちの父 (60 代、 工務店経営) に見せて 3 秒で『何屋か』分かる」
