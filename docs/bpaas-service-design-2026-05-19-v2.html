1. v1 (2026-05-19 朝) からの主要変更点

v1 は labor 試算と価格を主に推測で組んだ初版。 v2 は 競合 / 法務 / 税理士法 / LINE API の 4 軸並列リサーチを反映し、 v1 で見落としていた 10 件の論点を反映。

変更点内容
① 月末集中問題建設業の月末締めは業界慣習として確立 (建設業法 24 条の 3 でも月末締め前提)、 顧客側の締め日分散は期待しにくい → §5 で 5 つの対策案
② LINE 受信の物理的不可個人 LINE 経由は API 規約違反 + 画像 14 日期限。 LINE 公式アカウント (Messaging API) 必須、 Content API は即時 fetch + 自社 Storage 保存が前提 → §6
③ ANDPAD BPO 大手競合業界 No.1 ANDPAD (23 万社 / 68 万ユーザー) が BPO 提供中。 ただし「初期 ¥100K + 月額 ¥36K〜 60ID」 は 5-30 人規模に重く中小工務店向きでない可能性 → §3

⑥ 施主連絡代行は市場空白競合発見されず。 主顧客の「個人客 9 割」 (友人 N=1) に直撃する最重要サービス候補
⑦ 税理士法 52 条 OK/NG 線2025-05 警視庁初の名義貸し摘発 (3 年 / 300 件 / 報酬 1.3 億)。 推奨形態 = A 分業モデル (代行業者 + 提携税理士の 2 本立て契約) → §7
⑧ 個人情報保護法 2022 改正5,000 件以下小規模事業者除外撤廃で個人事業主も全件対象。 漏洩時 3-5 日速報 / 30 日確報、 罰金 100 万円 → §7
⑨ 損害保険必須FREENANCE 無料 + フリーランス協会 ¥10K + サイバー保険 ¥30-50K = 年 ¥40-60K を固定費に組込 → §8

4. サービス別詳細 (5 → 3 推奨に絞り込み)

① 現場写真整理代行 GO

BPaaS ¥18,000/月 → SaaS ¥7,000/月 (40%)

LINE で受信 → 工事番号別仕分け → 月末に PDF 台帳 3 テンプレ (完了報告 / 施主アルバム / 保証書添付)

v1 ¥15K から +¥3K: 蔵衛門 ¥3,300/月との差別化 (代行 + 工事番号自動推定 + 3 テンプレ自動生成) で +¥3K 正当化。 月 100-300 枚処理を契約上限化、 超過分は別料金

② 領収書・経理代行 GO

BPaaS ¥30,000/月 → SaaS ¥10,000/月 (33%)

領収書・請求書を LINE 写真送信 → OCR 仕訳 / 請求書発行 / 入金消込 / 月次レポート

⚠️ 税理士法 52 条 調査済: 仕訳・記帳代行 OK、 個別勘定科目判断・節税アドバイス・申告書作成 NG。 推奨 = A 分業モデル (代行業者 + 提携税理士の 2 本立て契約)。 認定支援機関税理士の事前確保が事業開始の前提条件。 詳細 §7

v1 ¥30K 据え置き: 業界相場 ¥6-30K の上限近辺。 仕訳 100-200 件 / 月想定で工務店特化プレミアム成立。 ただし建設業簿記の最低限知識 (完成工事高 / 完成工事原価 / 未成工事支出金 等 5 勘定科目) は習得必須

③ 日報作成代行 DROP / 保留

v1: BPaaS ¥20,000/月 → v2 で一旦保留

音声 LINE 送信 → Whisper 文字起こし + Claude 整形 → 日次/月末 PDF

⚠️ 競合不在 = 需要不在の罠:
  • 市場に「日報代行サービス」 が単独業務として存在しない (一般化していない)
  • 日報革命 / nanoty 等の SaaS はあるが、 工務店特化代行は発見されず
  • 友人 N=1 で「日報は紙」 と判明 (memory friend-hearing-result-2026-05-14)
  • 労力 5-13h で時給 1,500-4,000 円 = 5 サービス中で最低
判断: Phase 3 友人ヒアリングで需要確認後、 復活 or 撤退判定。 「日報書く時間がもう一現場分になる」 訴求が刺さるか、 そもそも日報書いてないかで判断

④ 見積書作成代行 DROP / 保留

v1: BPaaS ¥30,000/月 → v2 で一旦保留

過去見積 ベクトル DB 学習 → 新規案件 LINE 受信 → ドラフト生成 → 顧客確認 → 確定

⚠️ 機密性ハードルで外注文化なし:
  • 見積は粗利率を含むため工務店が外部委託しにくい (営業判断 + 機密性)
  • 建て役者 ¥4K/月 / アイピア ¥10K/月 等の SaaS で十分対応している市場
  • 初回オンボーディング 5-10h の負担が顧客にも masatoman にも重い
判断: 「テンプレ作成支援」 or 「初回学習だけスポット ¥50K」 にリフレーミングを検討、 月額 BPaaS としては撤退。 5/22 北原氏 Q12 で外注の心理障壁を確認

⑤ 施主連絡代行 GO ★ 最重要

BPaaS ¥25,000/月 → SaaS ¥8,000/月 (32%)

工事進捗写真 + 一言コメント受信 → 施主向け定期メッセージ作成 (週 1-2 回) → LINE/メール/SMS 送信 → クレーム兆候検出 → 工務店エスカレ

💡 v1 ¥20K → v2 ¥25K に値上げ:
  • 市場空白 調査済: 「施主への進捗連絡代行」 は単独サービスとして競合発見されず
  • ANDPAD 施主アプリは「自社で運用するツール」 で代行ではない
  • 友人 N=1 で「個人客 9 割」 判明 → B2C 接点肩代わりは主顧客痛点直撃
  • masatoman の本業 (受託) と最も近いスキル領域 = 文章生成・コミュニケーション
→ 5 サービス中で 最も差別化が効き値上げ余地がある
⚠️ クレーム責任分界 必須対応: 契約書で「代行範囲」「最終判断は工務店」「賠償上限 = 月額 1-3 ヶ月分」 明示 + サイバー保険必須。 詳細 §7-8

多軸比較の核心結論

  3. 「需要最高 × クレーム最高」 の交差点: 施主連絡は最重要 (S 優先) だが、 法務 + 損害保険 + 契約書整備が必須。 税理士法の経理と並ぶリスク TOP2
  4. 「学習コスト × 属人化」 が経理の最大壁: 建設業簿記 + 税理士法線引き + 提携税理士確保で参入障壁高、 一度乗り越えれば差別化堀になる
  5. 3 推奨セット (写真 + 経理 + 施主) の補完性: 競合密度 / 需要 / リスク のバランス良し。 単独販売ではなく 「セットで月 ¥73K」 で価値最大化。 単機能だけだと競合 (蔵衛門 / 記帳代行) に潰される
  6. 1 つだけ着手するなら写真整理: 実装最楽 + マーケ材料化容易 + クロスセル起点 + 法務リスク低 で MVP として最適。 ただし利益単独では細いので、 写真整理を「フック」 にして経理 + 施主連絡へ誘導するセット販売を前提に

5. 月末集中問題への対策

v1 の致命的見落とし: 「月末に PDF 配送」 設計だと 3 顧客契約で月末 5 営業日に 18-24h 業務集中 = 物理不可能。 建設業の月末締めは業界慣習として確立済 (建設業法 24 条の 3 が月末締め前提)、 顧客側の締め日分散は期待しにくい。

8.3 フリーランス新法 (2024-11 施行)

masatoman は工務店相手 = 受注側 (特定受託事業者)、 副業外注を使うと発注側 (特定業務委託事業者) にも該当。 両面影響:

  • 副業外注に発注時、 業務内容・報酬・支払期日等を書面 or 電磁的方法で明示
  • 副業外注への支払は受領日から 60 日以内
  • 2026-01 以降 振込手数料を外注負担にすると「報酬減額」 違反
  • 1 ヶ月以上継続発注で 7 つの禁止行為 (受領拒否 / 報酬減額 / 買いたたき 等)
  • 6 ヶ月以上発注で就業環境配慮義務 (妊娠 / 育児 / 介護 / ハラスメント)
  • 違反 = 50 万円以下罰金

10.3 BPaaS の「高価格」 が正当化される理由

主顧客 (IT 弱者の工務店オーナー) は「自分で操作したくない」 という核心特性 (memory customer-target)。 SaaS は学習コスト障壁で受け入れ困難 → BPaaS で「LINE に送るだけ」 という入口を作る → 一度慣れると SaaS に動かない (粘着性) → BPaaS が安定収益源として残る、 という戦略的価格構造。
