🥇 A. 写真整理代行 — 実装設計 + 月末対策
LINE Messaging API + 黒板 OCR + AI 仕分け + PDF 生成

アーキテクチャ

[現場監督 LINE] ↓ 写真送信 (現場黒板 + 工事写真) [LINE 公式アカウント] ↓ Webhook (即 200 + webhookEventId 重複排除) [Vercel Function] ↓ Content API で即 fetch (1 週間以内) [Supabase Storage] (顧客別 prefix) ↓ [AI 仕分けエンジン] - 黒板 OCR (工事名 / 工種 / 日付) - EXIF 補完 (撮影時刻 + GPS) - 工事番号自動推定 ↓ [PDF 台帳生成 (3 テンプレ)] - 完了報告書用 / 施主アルバム用 / 保証書添付用 ↓ [Reply Message で配送]

月工数 内訳 (3-5h / 月)

  • 写真受信 + 初期分類 (月 100-300 枚): 1.5-2.5h
  • 工事番号確定 + タイトル付け: 30 分
  • PDF 生成 (テンプレ化済): 30 分
  • 顧客 LINE 返信 + 配送: 30 分

月末集中対策

  • 「工事完了タイミング配送」 + 「週次 dashboard」 に分散 (月末固定をやめる)
  • 顧客契約時に 5/10/15/20/25/末 から締め日選択
  • 3 社でも月末ピーク 2-3h に抑える

競合との差別化

  • 蔵衛門 ¥3,300/月 (3 人) → 工務店オーナーが「自分で操作」 必要
  • ANDPAD カメラ (23 万社シェア) → UI 不満 + 写真消失で離脱層多い
  • ノンコア 時間チャージ ¥1,375-1,750/30 分 → 中小工務店には重い
  • masatoman = 「LINE 投げるだけで全部やってくれる人」

AI エージェント化への二段ロケット

  • Phase 1: BPaaS 月 ¥20K (3-5h 人手介在)
  • Phase 2: 黒板 OCR + AI 画像認識で 9 割自動化
  • Phase 3: AI エージェント SaaS 月 ¥5K-10K にダウングレード版を後付け

リスク

  • 個人情報保護法 — 施主の自宅外観 / 表札が写る画像は個人情報該当、 委託範囲明示必須
  • 初回学習データ蓄積 (1-2 ヶ月) で工事番号推定精度が安定するまでは人手介在比率高

4 業務領域

① 過去施主アフター
10 年分の施主リスト整理 + 連絡先取得 (家族 + 紹介ルート含む) → 不具合発生時の 1 次受け + 工務店エスカレ
② 定期点検リマインド
6 ヶ月 / 1 年 / 3 年 / 5 年 / 10 年 の自動リマインド LINE 送信 → 工務店から訪問日時連絡 → 完了報告
③ 見積後追客
見積提出後 1 週間以内に LINE 追客 → 反応に応じて電話 1 次受け → 「お盆明けに行く」 等の 放置防止
④ 季節挨拶 一括代行
年賀状 / 暑中見舞い / 工事完了 1 年記念 メッセージを LINE / メール / SMS で一括送信

月工数 内訳 (5-15h / 月)

  • 初期セットアップ (施主リスト整理): 5-10h (1 回のみ)
  • 定期リマインド送信 (週 1-2 回): 1-2h/月
  • 見積後追客 (案件発生時): 1-3h/月
  • 1 次受け対応 (返信時): 2-5h/月
  • クレーム エスカレ判断: 1h/月

AI エージェント化への道

  • Phase 1: 人手で施主リスト整理 + LINE Bot で定期リマインド自動送信 (BPaaS ¥30K)
  • Phase 2: 1 次受けを AI が応答 (Claude + ベクトル DB で過去対応参照)、 困難ケースのみ人手
  • Phase 3: 工務店向け CRM SaaS 月 ¥8-15K にダウングレード版を後付け

競合との差別化

  • 競合完全不在 — ANDPAD 施主アプリは「自社で運用するツール」、 代行は市場ゼロ
  • kintone / Salesforce 等の汎用 CRM → 工務店オーナー (IT 弱者) に届かない
  • 「過去 10 年の施主全部 LINE で繋がってます」 という状態を作る価値

単機能ではなくセットで価値最大化 — 競合 (蔵衛門 / ANDPAD) から守る

A 写真整理を「フック」 → B 顧客連絡へアップセル → F 議事録を Add-on の流れ。 主顧客 (IT 弱者) は「全部 LINE で送るだけ、 工務店業務 まるごと丸投げ」 を求める。 単機能 SaaS では届かない層に届く戦略。

セット販売の論理
単機能では蔵衛門 / ANDPAD に潰される
A 単独 ¥18K は蔵衛門 ¥3,300 と価格差 5-6 倍 → 営業困難。 セットなら「全部丸投げ」 という主顧客 (IT 弱者) の core 価値を満たす → SaaS では実現できないポジション。
月末集中対策は別資料 (bpaas-service-design-v2 §5) 参照 - 工程分散 + 15 日中間処理
