# 競合分析: 旧 3 案がダサかった理由と、 新方向

2026-05-11、 masatoman の「全部ダサい」 を受けて 7 サイト横断調査。

## 調査したサイト

| カテゴリ | サイト | 何を見るか |
|---|---|---|
| 個人 Web 屋 (masatoman 好み) | [hiraomakoto.jp](https://hiraomakoto.jp/) | テキストだけ、 装飾ゼロ、 高級活字本 |
| 個人 Web 屋 | [bokoko33.me](https://bokoko33.me/) | 作品カタログ、 ヒーロー画像なし、 ぼかし感 |
| 工務店 HP (建築家系) | [sakumastudio.com](https://sakumastudio.com/) | 明朝、 ヒーロー画像なし、 80 作品タイトル列挙 |
| 工務店 HP (Editorial) | [alain.co.jp](https://alain.co.jp/) | 哲学的 Hero、 顔写真 + プロフィール |
| 海外建築家 ポートフォリオ | OnderTak / Nicholas Gurney / Mi Zhou | 白基調 + アクセント 1 色、 写真主役 |

## 「ダサくない」 8 原則 (国内外で共通)

1. **白背景 + 黒テキスト** + アクセント 1 色だけ
2. **明朝系日本語フォント** (sakuma / alain)
3. **大きなキャッチフレーズ Hero** または **画像なし Hero**
4. **作品 = 主役**、 タイトル中心、 写真は最高品質のみ
5. **顔写真 + プロフィール** を Hero 直後に置く (信頼感の核)
6. **物語性のあるプロジェクト解説** (単なる施工例の上位)
7. **「無料相談」「相談会」** で敷居を下げる
8. **専門性に絞る** (alain 風 「半径 5km + 狭小住宅」)

## 旧 3 案がダサかった理由 (推測)

| 旧 3 案の共通問題 | 原則違反 |
|---|---|
| Hero → Problems → Examples → Pricing → Contact の典型 BtoB LP 構成 | ❌ alain は「哲学的 Hero」、 sakuma は「ニュース型 Hero」 で先に固定構造を崩している |
| brown + ベージュの warm editorial 全面 | ❌「白 + 黒 + アクセント 1 色」 が原則。 brown を全面に塗ると「印刷物」「俱楽部チラシ」 化 |
| 抽象 SVG イラストカード | ❌ 「写真は最高品質のみ」 が原則。 イラストは Web 屋臭くなる |
| 価格バッジ・効果数字バッジ | ❌「LP 感」 が前面化。 sakuma / alain は数字を Hero に出さない |
| About / 顔写真 が後半 or 仮アバター | ❌「顔写真 + プロフィール = 信頼感の核」 を後回しにしている |
| 「ANDPAD より安い」「ANDPAD を入れるほどじゃない」 系のコピー | ❌ 比較を主軸にしている = 「Web 屋っぽい」。 alain は他社比較を 1 行も書いていない |

要するに **「BtoB SaaS LP」 的なノリ** が masatoman の感性 (hirao 系) と工務店オーナーの「Webっぽくない静かな信頼」 の両方を裏切っていた。

## 新方向: **Editorial Studio** (主軸 1 案)

**コンセプト**: 「建築事務所の HP」 の佇まいで、 工務店向け Web 屋を表現する。 alain × sakuma × hirao のハイブリッド。

### Hero (画像なし or 極小)

```
□  Masato Works                                    [ Contact ]

                                                        (大きな余白)


           工務店の流れを、

           そっと  作り直す。



           ─ 5〜30 人の地域工務店向け
             ホームページと業務改善を 一人で


                                                        (大きな余白)
```

- 配色: 白 `#fafaf7` 基調、 テキスト `#1f2a37` (ほぼ黒)、 アクセント `#7a5c38` (細線と小見出しだけ)
- フォント: 大見出しは Noto Serif JP **45-72px**、 行間 1.55、 余白 200px 級
- Hero 画像なし。 もしくは右下に **小さな空間** が浮かぶ 1 枚 (具体的な「現場」 ではなく抽象的)
- CTA は 「Contact」 だけ、 右上に控えめに

### Section 1: About 即出し (Hero 直後)

alain.co.jp の最大の発明 = **顔と物語を Hero 直後に置く**

```
□  About


   井原 誠斗  Ihara Masato
   一人で全部やっている Web 屋です。


   工務店向けの ホームページから 日報・写真整理・領収書まで、
   現場と事務の流れを 一人で組み上げます。
   ANDPAD を入れるほどじゃない 5〜30 人の工務店、
   個人親方、 一人で全部見てる現場監督が 主なお客様です。


   [ 顔写真 :: 実写、 仕事中 ]
```

- 顔写真は 320px 四方くらい、 円形ではなく **長方形** (写真展の作品っぽく)
- プロフィールは 4-5 行、 「全部やる」 と「主なお客様」 を 1 段落で

### Section 2: Works (sakuma 風タイトル列挙)

```
□  Works


   01    車で しゃべるだけで 日報を作る             →
   02    現場写真を 自動で仕分ける                  →
   03    施主向け 工程進捗ページを 自動更新           →
   04    領収書を撮るだけで 経費に                  →
   05    図面を スマホで 一発呼び出し              →
   06    協力業者の見積確認を 軽くする              →
   07    現場連絡の 抜け漏れを 減らす              →
   08    工務店ホームページの 作り直し             →

                                  ─ すべて触れるデモです
```

- タイトル + 番号 + 矢印だけ、 1 行ずつ
- hover で 行が薄く伸びる、 写真は **個別ページ** で初めて出る (sakuma 流)
- 「触れるデモです」 を控えめにフッターに

### Section 3: 哲学 / なぜ (alain 風)

```
□  Philosophy


   「全部やる」 ことが、 工務店の現場には 一番合う。


   現場と事務、 ホームページの「ちょっと面倒」 は、
   ぜんぶ繋がっています。
   バラバラの会社に頼むと、
   結局 連絡係に 時間が消えます。

   一人の作業所 みたいなところで まとめて頼める。
   それが この仕事の あり方です。
```

- 「ANDPAD より」 とか「価格は」 とかは書かない
- 哲学だけ語る、 数字は出さない

### Section 4: Pricing (控えめ 1 段組)

```
□  Pricing


   初期 10 万円 + 月 1 万円 から
   内容により ご相談


   初回のご相談は 無料です。
```

- 4 ティアは廃止、 1 行で済ます (sakuma も 1 行)

### Section 5: Contact

```
□  Contact


   ご相談を お待ちしています。


   [ LINE で 相談する ]    [ メールで 相談する ]


   電話・LINE・メール、 どれでも 大丈夫です。
```

- LINE と メール 並列、 LINE を主訴求 (緑) にせず 同列扱い
- 「30 秒」「30 日」 などの煽り数字は **使わない**

### Footer

```
─────────────────────────────────────
© 2026 井原誠斗 (Ihara Frontend)
Portfolio  ・  Lab
─────────────────────────────────────
```

## 派生案: **Minimal Magazine** (補欠、 写真主役)

主軸案より少しだけ「雑誌風」 の階層を入れる。

- Hero に **大判モノクロ写真** (工務店現場、 手元) + 一文だけ
- Works を「特集 + 補欠」 にする (1 件だけフル幅、 残り 7 件は小さく)
- 月別 / カテゴリ別の **記事構造** を入れ込む (将来ブログ化可)

主軸案より「写真の質」 に依存するので、 Gemini nanobanana2 で良い画像が出るかが鍵。 旧案 2 (写真主役) のパワーアップ版。

## 何が変わるか (旧 3 案との対比)

| 軸 | 旧 3 案 (どれも) | 新 Editorial Studio |
|---|---|---|
| 配色 | brown + ベージュ全面 | **白基調 + 黒 + brown 細線** |
| Hero | 価格バッジ / 写真 / 問いかけ | **画像なし、 余白最大の哲学的キャッチ** |
| About 位置 | 後半 or 仮アバター | **Hero 直後、 顔写真 + プロフィール** |
| Works | グリッドカード (画像付き) | **番号付きタイトル列挙、 sakuma 流** |
| 価格 | 4 ティア横並び | **1 行**「初期 10 万 + 月 1 万 から、 相談」 |
| 比較訴求 | 「ANDPAD より」 多用 | **競合に触れない、 哲学だけ** |
| CTA | LINE 緑大ボタン | **「ご相談」 単一、 装飾排除** |
| 全体の印象 | BtoB SaaS LP | **建築事務所の HP** |
