## 4. 5 つの隙間 (統合分析)

### 隙間 1: ANDPAD 代替 軽量版

**痛み**: 重い (Apple Store 2.9/5) / 高い (初期 10 万 + 月 3.6 万) / 職人が使えない (代行入力問題) / オフライン非対応

**空白**: 「職人が自分のために使いたくなるシンプルツール」

**裏付け**: 105 件中 ANDPAD 名指し不満 4 件 + 「代行入力」常態化 (E:85) + 通信環境問題 (E:78, S:80)

### 隙間 2: 写真台帳シンプル ⭐最有力

**痛み**: 写真台帳・日報が深夜残業の主因 / 蔵衛門「無題アルバム」の山 / Photoruction ボタン誤爆

**空白**: 「撮って自動で台帳になる」だけのシンプルツール。既存 3 社 (ANDPAD / 蔵衛門 / Photoruction) 全部に UI 不満

**裏付け**: E×S 全 105 件中トップ (7,360) / SaaS 導入で 1/4 削減事例 = ROI 実証済 / Solvability 75-80 (技術的に解決容易)

**v8.0 との対応**: **#03 (写真管理) と完全一致**

### 隙間 3: 営業 CRM / 追客

**痛み**: 見積提出後の音信不通 = タダ働き (E:82) / 紹介依存崩壊で年商 2-3 億 → 1 億割れ (E:72)

**空白**: ANDPAD/KANNA は「施工管理」、受注前の追客 → 見積 → 契約の CRM が完全空白

**裏付け**: 105 件中営業系 12 件 (11%) / 二代目が WEB 集客で代替できず不安を抱える

**v8.0 との対応**: #04 (顧客進捗) が近いが、隙間は「見積 → 受注前」寄り

### 隙間 4: OB 管理 / アフター対応

**痛み**: 引き渡し後のアフター対応放置 (E:88, S:55)

**空白**: 施工完了後の定期点検 → 修繕 → リピート受注の管理

**裏付け**: E:88 は全 105 件中 3 位の感情スコア。施主の信頼崩壊ポイント

**⚠️注意**: 一次情報根拠なし (Claude 推論 3 層、5/25 demo-role-redefinition で自覚済)

### 隙間 5: 補助金申請ナビ

**痛み**: 公式ページが難解 (E:70, S:70) / jGrants 3 時間 timeout 全消失 (E:78, S:65) / 採択率 43.8% 急落

**空白**: 「あなたの会社、この補助金使えます」の対象判定 + 申請ナビゲーション

**裏付け**: 105 件中 20 件 (19%) が補助金・行政手続き系。IT 導入補助金の年間採択 70,742 件 = 巨大市場

**v8.0 との対応**: 候補外 — **新発見。ヒアリングで検証価値大**
