v3 → v4 戦略大転換 v4 BPaaS (代行) → 覚えなくていい DX (運用支援 + 必要時代行)

現場ヒアリング (友人 N=1 + 業界観察) で 工務店は「費用」 より「学習コスト」 を嫌う ことが判明。 v3 の「BPaaS = 全部代行」 中心思想は 完全代行リスク 6 つ を抱え、 工務店側が拒否反応を示す可能性が高い。 v4 で 「DX 運用支援」 に方向転換。

❌ v3 BPaaS の問題 (完全代行リスク)

  • 社内知識が蓄積されない → 解約時に戻せない
  • 属人化 (masatoman 依存)
  • 「仕事を奪われる」 懸念 (奥さん経理問題)
  • 社員が運用を覚えない → DX が根付かない
  • 「AI で人削減」 訴求は工務店に刺さらない
  • 顧客ごとの重カスタム → スケールしない

工務店の本音: 「人を減らしたい」 ではなく「楽になりたい」

⭕ v4 「DX 運用支援」 = 覚えなくていい DX

新ポジ: DX ツール販売会社 ではなく DX を最後まで運用できるよう支援する会社

v4 軸メッセージ (3 本):

  • 「LINE 送るだけ」
  • 「覚えなくていい」
  • 「必要ならこちらでやる」

顧客から見た masatoman: 「AI 会社」 ではなく 「DX の面倒を一緒に整理してくれる人」

工務店の本当の痛み = 「DX 運用疲れ」

工務店が嫌なのは 入力 / 学習 / ルール化 / 写真整理 / 通知地獄 / 情報散乱 / LINE 管理。 つまり「DX ツール不足」 ではなく「DX 運用疲れ」。 ANDPAD を導入していても使いこなせていない (#22 感情度 92 = 「死ぬほど苦痛」)、 一度詰まると触らなくなる、 教える人がいない、 という現実が背景。
