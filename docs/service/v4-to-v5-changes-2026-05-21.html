// 2. v4 → v5 への転換の理由

v4 公開直後 (5/20 朝) に判明した致命的問題:

  • 主顧客 (60-70 代 IT 弱者社長) は LINE すら使えない。 「LINE 送るだけ」 訴求が刺さる相手と、 主顧客像が矛盾
  • 友人 N=1 ヒアリング結果「メール 3 割使えない」 と整合しない
  • 建設業界の通信実態を見ると FAX / 電話 / 郵送 / 訪問が業務の中心
  • masatoman の指摘で気付いた: 「住む場所によって戦略変わらないよね?」 = LINE 主軸自体が地理ではなく 顧客像との整合性 の問題

v5 への転換 (5/20 夕) で何が解決したか:

  • 顧客像 (60-70 代 IT 弱者) と訴求 (FAX 主軸) が完全整合
  • 業界の既存通信構造 (FAX / 郵送) に乗る = 学習コストゼロ
  • FAX バイラル拡散戦略 = 業界の通信構造を逆手に取った独自差別化軸を発見
  • 地理依存ナシ = 五反田引越し計画にも対応可能
  • 競合 5 社 (ANDPAD BPO 等) が見えていない領域 = ブルーオーシャン化

// 3. 具体的な改善内容 6 項目

① 「LINE 送るだけ」 訴求の撤回 → 「FAX / 郵送 / 電話 で OK」

主顧客 (60-70 代) に LINE は届かない。 FAX / 電話 / 郵送が業務の中心。 既存通信手段に合わせることで学習コストをゼロにする。 「いつもどおりで OK」 という新メッセージが追加された。

② Hero Statement「ANDPAD で詰まった工務店向けに...」 を新規追加

v4 では明確な 1 行訴求がなかった。 v5 で 競合 5 社 (ANDPAD BPO / ツクノビ事務 等) との差別化を 1 行で説明できる Hero Statement を確立。 北原氏面談で冒頭から事業像を掴める構造に。

③ FAX バイラル拡散戦略 = 業界の通信構造を逆手に取る独自軸

「LINE 主軸」 では 1 対 1 で閉じるが、 「FAX 主軸」 だと 1 通の FAX が業者間で 5-10 人に回覧される文化 に乗れる。 これが競合 5 社が見えていない領域 = ブルーオーシャン化。 営業コストゼロで認知拡散。

④ 「地理依存ナシ」 = 五反田引越し対応 + 全国対応の可能性

v4 では「多摩限定 + たましん信用補完」 を差別化軸として固定していたが、 引越し計画判明で柔軟性が必要に。 v5 では「FAX 主軸 = 地理依存ナシ」 を新たな差別化軸として確立、 拠点変更にも耐える設計。

⑤ 「DX しなくていい」 反主流ポジ = 業界紙取材材料として強力

v4 の「覚えなくていい DX」 は DX 推進ポジに留まる。 v5 の「DX しなくていい」 は 反主流ポジ として業界紙の取材ネタとして異常に強い (新建ハウジング / 建築知識 / 日経ホームビルダー 等)。 信用補完レイヤー B の獲得経路。

⑥ 物理オペ参入障壁 = 誰もやりたがらない領域こそ堀

FAX / 郵送主軸は「誰もやりたがらない物理オペ」 を含む。 大手 BPO (ANDPAD BPO 等) はデジタル前提でこの領域に参入してこない。 これが 競合に対する圧倒的優位 として機能する。

v5 でも残った課題 + 引越し対応 + 戦略凍結

v5 で完璧ではない部分 + 5/22 北原氏で確認したい点

// 5. 引越し対応の改善 (5/21 追加)

5/21 朝、 五反田引越し計画判明: v4 では「多摩限定」 を差別化軸として強調していたが、 引越しで使えなくなる懸念。 5/21 議論で masatoman の指摘「住む場所によって戦略変わらないよね?」 = 商品設計の本質 (FAX 主軸 = 地理依存ナシ) を理解した結果、 v5 は引越しに耐えると確認。

差別化軸の置き換え:

  • 旧 (v4): ② 多摩地区限定 + たましん信用補完
  • 新 (v5): ② 地理依存ナシ + 引越し対応

これにより、 多摩既存顧客は遠隔運用継続 + 五反田引越し後の城南エリア新規開拓 + 全国展開の可能性、 すべてが同じ商品設計で対応可能に。
