3. **認定支援機関税理士 / 業界紙記者 / 補助金窓口** など、 対外的な「旗」 を見せる相手が今後増える
4. **N=1 で前提崩壊 5 件** ([[friend-hearing-result-2026-05-14]]) があり、 戦略は揺れている。 だからこそ **揺れない思想軸** を先に固める意味がある

## A. ビジョン 3 案 (5-10 年後の到達点)

### 案 V-1: 「DX で建設業の残業を 0 に」 ★ masatoman 提案

**理由 / メリット**:
- 行政語「DX」 = 北原氏 / 補助金窓口 / 認定支援機関税理士 / 業界紙 に通じる公式語
- 「残業を 0 に」 = 言い切りで強い、 ビジョンとして to-be が明確
- 「建設業」 = 工務店だけでなく上流 (建材屋・設計事務所) も射程内、 [[komuten-channel-strategy]] と整合
- 補助金 / 業界紙取材ルート ([[industry-press-authority-borrow]]) で繰り返し使える

**リスク / 注釈**:
- 「ANDPAD = DX」 既視感あり、 差別化が本文側で必要
- 「残業」 が真の課題か N=1 確証なし。 友人ヒアリングでは「日報は紙のまま OK」「残業 1 時間」 程度の言及。 主訴は「残業」 ではなく「個人客対応」 や「紙とデジタルの架け橋」 かもしれない → 5/22 + 雪だるま 2-3 件 で N=4-5 検証後に「残業」 が真の主訴か再確認

### 案 V-2: 「DX で建設業の 手間 を 0 に」

**理由 / メリット**:
- V-1 の「残業」 を「手間」 に置換。 N=1 で残業が主訴と確証ない不確実性を吸収
- 「手間」 = 残業 + 紙作業 + 電話往復 + 探し物 を全部包含する広い概念
- 個人客 9 割 + 紙文化の主顧客実態 に揺らがない

**リスク / 注釈**:
- 「手間 0」 は「残業 0」 より弱く聞こえる (数値で測りにくい)
- 行政・補助金窓口的には「残業 0」 の方がインパクトある

### 案 V-3: 「DX で建設業の毎日を 1 時間 取り戻す」

**理由 / メリット**:
- 定量的 (1 時間) で測りやすい、 KPI に落ちる
- 「残業 0」 のような言い切りより現実的、 過剰約束にならない
- 1 時間 = 1 日 / 8 時間労働の 12.5%、 まあまあ大きい数字でかつ達成可能
- masatoman さんの「撤退判断早い」 価値観 (現実主義) と整合

**リスク / 注釈**:
- 表現が長め、 ビジョンとしてのキャッチーさがやや弱い
- 「1 時間」 の根拠を聞かれる (friend hearing N=1「事務所での残業 1 時間をなくす」 = voice-daily 旧訴求の文脈、 N=4-5 で再確認必要)

## B. ミッション 3 案 (なぜ存在するか / 永続)

### 案 M-1: 「アナログのまま、 工務店の毎日を整える」

**理由 / メリット**:
- 主顧客 (IT 弱者 工務店オーナー) 向け平易語 ([[customer-target]] 整合)
- 「アナログのまま」 = 紙 / 電話 / LINE / ファックス で生きてる主顧客に寄り添う姿勢
- N=1「メール使えない 3 割」「日報は紙」 を肯定する出発点
- ANDPAD 既存ユーザの不満「使いこなせない」 への代替提示

**リスク / 注釈**:
- 「アナログのまま」 は「DX で」 ビジョンと矛盾するように読まれる可能性 (実際は「無理に置き換えず、 必要な所だけ DX」 という意味)

### 案 M-2: 「現場と事務の間を、 1 人窓口で整える」

**理由 / メリット**:
- 現状サイトのコピー (「現場と事務の流れを、 落ち着いて整える」) と地続き、 既存資産活用
- [[ai-positioning]] の「ひとりで全部作れる職人」 と整合
- 工務店オーナーの「色んな業者と話すの面倒」 に刺さる

**リスク / 注釈**:
- 「1 人窓口」 = スケールしない印象。 将来複数人体制になると矛盾する
- ただし当面は masatoman 1 人なので嘘ではない

### 案 M-3: 「個人のお施主様も、 工務店さんも、 両方が楽になる仕組みを」

**理由 / メリット**:
- friend hearing N=1「個人客 9 割」 を真正面から反映
- B2B + B2C ハイブリッド戦略 (v3 書換ポイント #3) を思想化
- 「両方が楽」 = 工務店経由で個人施主にも届く構造を明示

**リスク / 注釈**:
- N=1 段階で個人客 9 割を強調するのはやや早い、 5/22 Q8 で北原氏に「中堅工務店も個人客中心ですか?」 確認後の方が安全
- 主顧客が「個人客対応」 を主訴と思ってない場合、 ピントずれ

## C. バリュー 7 案 (5 つ選んで採用)

> 行動指針 / 判断基準。 商談・採用・PR で繰り返し使う

### 案 X-1: 「現場の言葉で話す」

- 出典: [[customer-target]] (IT 専門語全廃方針)
- 適用シーン: 営業 / 提案書 / サイトコピー
- 強み: NG ワード違反を全て吸収するメタルール

### 案 X-2: 「電話 1 本で動く」

- 出典: friend hearing N=1「メール使えない 3 割」
- 適用シーン: 商談導線 / 連絡 SLA
- 強み: 「DX 屋なのに電話?」 のギャップで記憶に残る

### 案 X-3: 「派手より着実」

- 出典: [[design-direction]] (warm editorial、 hiraomakoto.jp 系)
- 適用シーン: デザイン判断 / 提案規模選択 / 価格設定
- 強み: 工務店オーナーの「いきなり大きく変えさせようとしてくる業者」 警戒心への対応

### 案 X-4: 「紙と LINE のまま、 使える」

- 出典: friend hearing N=1「日報は紙」「個人客は LINE」
- 適用シーン: SaaS 設計 / 商談時の説明
- 強み: ANDPAD 系の「全部置き換え」 思想との差別化

### 案 X-5: 「数字より一次情報」

- 出典: STRATEGY §2「真の弱み = 作る前の調査不足」 + N=1 で前提 5 件崩壊した経験
- 適用シーン: 戦略判断 / SaaS 機能優先順位
- 強み: masatoman 自身の北極星、 安易な PMF 早期化を防ぐ

### 案 X-6: 「個人客 9 割の現実から目を逸らさない」

- 出典: friend hearing N=1
- 適用シーン: 商談 / SaaS 設計 / マーケ
- 強み: B2B SaaS 系業者と差別化、 工務店オーナーの本音に寄り添う
- リスク: N=4-5 確認後に「9 割」 が間違いと判明したら撤回必要

### 案 X-7: 「ANDPAD を否定しない」

- 出典: friend hearing N=1「ANDPAD 使いこなせてない、 何が足りないかもわからない」
- 適用シーン: 商談 / 競合への姿勢 / 業界紙取材
- 強み: 業界エコシステムを敵に回さない、 共存ポジション
- ただし「ANDPAD のサブ」 になる危険性もあり、 表現の調整必要
